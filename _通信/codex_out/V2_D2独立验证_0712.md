时间：2026-07-11 18:45（America/Chicago）  
范围：独立扫描仓库 ST 源码中的分类函数、策略表消费者和 `rAlpha/rBeta/rGamma/rDelta` 赋值点，再对照 Claude 的 D-2。未运行 Automation Builder，未运行 `tools/ab_scripting`，未修改现有代码。

## 独立结论

**同意 D-2 的核心判断：PLC 的场景自适应权重“表在、线没接”，生产调用链从未按场景消费该表。严重度“阻塞·报告级致命”成立。**

但旧版 D-2 文字需要两处精确修正：

1. `plc/08_GVL_WeightPolicy_generated.st:8-14` 是有效的 `VAR_GLOBAL CONSTANT` 数组声明，不是注释；只有 `:16-19` 的“用法/赋值”处于 `(* ... *)` 注释内。应称“活常量无消费者”或“数据表未接入”，不应称“`:9-19` 整段被注释”。
2. PLC 权重可由管理员/HMI手动改，`FB_ParamGuard` 会把管理员当前值保存为新基准。因此最严谨的结论不是“运行时永远固定默认值”，而是“没有自动的场景相关赋值；若无人手动改参，则沿用初始化默认值 α=1.0、β=0.6、γ=0.4、δ=0.15”。手动调参不等于场景自适应。

## 原始证据链

### 1. 分类函数不存在

全仓检索没有任何函数体：

```powershell
rg -n --hidden -S '^\s*(FUNCTION|FUNCTION_BLOCK)\s+FC_ClassifyScene\b' F:\abb_wh_work
```

结果：`NO_MATCH`。

`FC_ClassifyScene` 在生产源码中的唯一提及是 `plc/08_GVL_WeightPolicy_generated.st:5` 的说明注释。其余命中均为 docs、任务信箱或生成器字符串，不是 IEC ST 函数实现。

### 2. 权重表存在，但没有生产消费者

- `plc/08_GVL_WeightPolicy_generated.st:8-14`：`aPolicyDelta` 与 `aPolicyBG` 是有效的 3×3×2 常量数组。
- `plc/08_GVL_WeightPolicy_generated.st:16-19`：唯一的查表赋值写在注释示例中，不会被编译执行。
- 对全部 `plc/*.st` 检索 `aPolicyDelta|aPolicyBG|iDensity|iDist`，除上述声明、注释和注释中的示例外没有任何命中；`PRG_Main` 没有权重查表或权重赋值。
- `sim/adaptive_weights.py:152-155` 生成 ST 文件时主动给“用法”加上 `(* ... *)`，所以重新生成也会继续产出未接线的示例，而不会生成可执行调用。

可复核命令：

```powershell
rg -n -S 'aPolicyDelta|aPolicyBG|iDensity|iDist' F:\abb_wh_work\plc -g '*.st'
rg -n -S 'GVL_Param\.r(Alpha|Beta|Gamma|Delta)\s*:=' F:\abb_wh_work\plc\05_PRG_Main.st
```

第二条结果为 `NO_MATCH`。

### 3. 四个权重的真实来源与赋值点

- 初始化：`plc/02_GVL.st:53-56` 给出 α=1.0、β=0.6、γ=0.4、δ=0.15。
- 消费：`plc/03_Functions.st:212-213` 的 `FC_CalcScore` 每次直接读取 `GVL_Param` 当前值；这里没有场景判档或查表。
- 守卫快照：`plc/04_FB_Warehouse.st:1078-1094` 在管理员在位时读取当前权重并刷新快照。
- 守卫回滚：`plc/04_FB_Warehouse.st:1095-1110` 在非管理员级把权重写回已保存快照；`:1104-1105` 不是自适应赋值。
- 测试：`plc/06_PRG_Test.st:442-449` 只临时修改 `rDelta` 验证权限守卫，最后恢复 0.15；不是生产场景逻辑。
- 注释示例：`plc/08_GVL_WeightPolicy_generated.st:17-19` 的 β/γ/δ 查表赋值不执行。
- `rAlpha` 没有任何场景赋值；它只在初始化、评分读取和 ParamGuard 快照/回滚中出现。

全仓 ST 赋值检索：

```powershell
rg -n -S 'rAlpha\s*:=|rBeta\s*:=|rGamma\s*:=|rDelta\s*:=' F:\abb_wh_work\plc -g '*.st'
```

没有发现隐藏的场景相关生产赋值。

### 4. 活跃的 FB_SceneDetect 不能反证 D-2

`plc/05_PRG_Main.st:193-196` 确实每周期运行 `FB_SceneDetect`，自动模式下把 `iRecBatch` 写入 `SelStrategy`。但这条链只切换 `near/AWRA-LS` 等策略；`plc/05_PRG_Main.st:188-210` 没有写入四个评分权重，也没有读取 `aPolicyDelta/aPolicyBG`。因此：

- A2“场景→策略推荐”是活代码；
- A1/E v2“场景→评分权重查表”未接入；
- 两者名称相近，不能混为同一个自适应机制。

## 建议的最小准确措辞

> PLC 已包含与 sim 同源的 `aPolicyDelta/aPolicyBG` 常量表，但没有 `FC_ClassifyScene` 实现，且唯一查表赋值示例位于注释内；`PRG_Main` 没有消费者。因而 AC500 当前不会随场景自动改权重，默认运行使用 α=1.0、β=0.6、γ=0.4、δ=0.15（管理员仍可手动改成另一组静态值）。

## 附带发现

- 生成器本身只生成“注释用法”，说明缺口不是单次漏粘贴，而是当前生成链设计就没有产出可执行接入代码。
