## 一、故事线导览六章

### 1. 赛题：仓位不是“有空就放”

**章节标语：让每一次存取，都从正确的位置开始。**

20×20 立体仓库看似只是 400 个格子，实际同时受重量承载、访问频次、货物体积、预占位置和堆垛机运动约束。我们的任务不是画出一个整齐货架，而是在可行性为零容错的前提下，让高频货更近、重货更低、路径更短，并把结果落实为 ABB AC500 可执行的结构化控制逻辑。

出处：`docs/赛题原文_2026智储优控.md`“任务要求及作品交付”；`docs/canonical_assumptions.md` A1-A9、B1-B4。

### 2. 建模：先把现实边界写清楚

**章节标语：每个数字都能追到假设，每个假设都能接受质疑。**

模型采用单巷道、单机、双轴同动的切比雪夫行程时间，并显式加入速度、加速度、层承重、体积、预占和装卸边界。报告主口径开启梯形加减速；所有随机实验固定 seed、各策略使用同一批货物。官方给定、官方允许自设和项目假设分栏登记，避免把方便计算的设定伪装成题面事实。

出处：`docs/canonical_assumptions.md` A4-A8、B1-B6；`docs/报告草稿_B1B2B10_0711pm.md` B2“局限性与扩展”。

### 3. 算法：目标不同，正确策略也会不同

**章节标语：不是押注一个万能算法，而是让目标档决定取舍。**

AWRA-LS 综合效率、稳定与能耗；CB/TOB 分别服务高密度与周转优先场景，检测器按场景画像推荐策略。同一 20 件在线批中，词典序/TOB 为 122.0s、222m，holistic/AWRA 为 186.0s、370m：前者追求时间，后者支付时间溢价换多目标护栏。sim 词典序档在可比域达 oracle 98.5%，该比例不是 PLC 实测或理论上限。

出处：`docs/报告草稿_§6可视化_0712.md` §6.3；`docs/报告草稿_B1B2B10_0711pm.md` §3.2-3.5；`sim/out/oracle_gap_summary.txt`。

### 4. PLC 实现：把算法翻译成可审查的工业程序

**章节标语：算法不是黑盒脚本，而是能逐块检查的 ST 执行链。**

核心策略、场景检测、参数守卫、权限、检修位、统计与三页 HMI 均以 ST 功能块和状态机组织。CB/TOB 与 Python 参考实现由黄金向量逐位锁定，AB PC仿真最近一次在线读回为 75 项通过、0 失败。75/0 证明当前源码与证据域内断言成立，不代表实体控制器时延、全量压测或网络闭环已经完成。

出处：`docs/canonical_assumptions.md` C11；`plc/04_FB_Warehouse.st`、`plc/05_PRG_Main.st`、`plc/06_PRG_Test.st`；`tools/ab_scripting/logs/runtest_result_20260712_111501.txt`。

### 5. 物理验证：成功画面和失败边界一起交代

**章节标语：负结果不是删掉，而是用来划清下一步。**

Factory I/O 以 54 格代表性场景做 Python 直控技术预演，H5 记录到 cell 1/30/54 各三次、共 9/9 全链运行成功；独立复核同时指出根因唯一性、异常分支和实体位置反馈仍有缺口。VirtualAC500 的 Modbus TCP Client 在当前环境连接前同步失败，因此项目不把该结果升级为控制器网络闭环，直驱证据留待 PM5650 实机。

出处：`docs/overnight2-report_0712night.md:187,254`；`_通信/codex_out/0714_F3独立复核.md`；`_通信/codex_out/0713_F5二审.md`。

### 6. 结果与诚实边界：高分来自可复现，不来自放大措辞

**章节标语：能复现的成绩，才经得住评委拆解。**

30-seed untouched final test 给出 AWRA-LS 8.17±0.67s、较顺序基线降 61.6%、30/30 seed 正收益、违规与失败为 0，均为 sim H 泛化主证据。5-seed 8.40±0.48s 只作回归锚。吞吐 230 次/小时是行程-only sim 容量，年化能耗是净机械功情景换算；两者都不是现场电表或生产节拍承诺。

出处：`docs/canonical_assumptions.md` C9、C12；`sim/final_test_30seed.py`；`sim/out/final_test_30seed.csv`；`docs/报告草稿_B1B2B10_0711pm.md` §1.1。
