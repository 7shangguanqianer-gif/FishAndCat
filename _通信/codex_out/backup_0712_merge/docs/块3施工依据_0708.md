### §3b Modbus TCP Server 施工配方(0710 定向枪,sonnet;证据等级逐条标注)

1. **设备树**:右键 ETH 口 → Add object → **"Modbus TCP/IP Server"**(每 ETH 口限 1 实例,可与其他
   协议并存;协议栈固件自带,不装库)。★官方一手:help.plc.abb.com
   `Configuration of Modbus TCP_IP Server.html`;交叉验证=官方示例工程 AE-0980。
2. **Server 参数**:Port 默认 502;Byte Order 默认 Big Endian;Startup Behaviour 保持默认
   **Active**(随 PLC 运行自动生效,零 FB 调用;设 "No activity" 才需要 ModbusTcp 库的
   ModTcpServOnOff 手动启停)。★官方一手。
3. **变量暴露=%M 地址直映射**(核心机制,非映射表):变量必须在 GVL 里**显式 `AT %MW..` 声明**,
   %MWn ↔ Modbus 保持寄存器 n("%M memory addresses map directly to Modbus word addresses")。
   普通声明(无 AT)编译器自动分配地址,Server 读不到。★官方一手。
   → **对我方的设计含义**:GVL_Visu 现有变量均无 AT 声明。**推荐做法=新建独立 GVL_Modbus 镜像区**
   (`aMbSlotState AT %MW0 : ARRAY[0..19,0..19] OF INT;` + 标量 %MW400 起),PRG_Main 每周期整赋
   拷贝——零侵入现有变量/测试,还顺带解决"快照原子性"(拷贝瞬间即打包窗口)。
   ⚠️ 属 ST 改动:**Codex 画面施工完、ab_sync 可用后才动**。
4. **两个待实测点(推断级,上正式数据前先用递增已知值核验)**:①`ARRAY[0..19,0..19]` 整段连续
   占址与线性化顺序(IEC 惯例=行优先/最后一维连续,但无官方逐字);②REAL(%MD,占 2 寄存器)的
   字序,pymodbus 解码 Endian 参数以实测为准。
5. **读取拆包**:FC03 上限 125 寄存器(★官方一手 modbus_tcp_description.html)→ 400 INT 拆 4 次
   (100×4);并发无虞(单 Python 客户端 vs PM56xx 系列 15~50 并发 Server 连接上限,
   ★数据手册 3ADR010079 经代理提取,关键数字用原 PDF 复核)。背靠背请求已知坑:查无
   (ABB 论坛 3 个对口帖全数连接失败,如实记录为查不到,非"无坑"证明)。
6. **参考件**:官方示例工程 **AE-0980(3ADR010980)**《AC500 V3 Modbus TCP - Communication with
   all supported Function Codes》(GVL 声明写法+Server 挂载,library.e.abb.com 公开 PDF);
   技术总览 **3ADR010810**《Modbus TCP in AC500 V3》——⚠️ 其中 CI521/CI522 的"每设备 0x100 固定块
   自动映射(%M 4096-11007)"是**另一套机制**,与我方"任意用户变量映射"路径无关,读时勿混。

### §3c OPC UA 备选通道详情(0710 定向枪,sonnet)

- **App Note 编号确认**:3ADR010661《AC500 V3 and OPC UA - Configuration and Handling》rev D
  (PDF 为图片/压缩流,正文表格机读失败,如实记录;配置与性能数字改从官方在线帮助逐字取得)。
- **性能(★官方一手,help.plc.abb.com 两页交叉一致)**:`ItemMinSamplingRate=100ms`(客户端设 0
  也按 100ms=10Hz 上限,画面动画够用)、`SessionsMax=100`、`SecureChannelsMax=50`、
  订阅数无硬上限(受 CPU 性能约束);**单次 service ≤100 operations**(标题级)→ 400 元素
  拆 400 节点一次 Read 会撞墙,**必须整数组一个节点读/订阅**。
- **数组整节点行为**:CODESYS 默认数组=一个 symbol(勾"Export array entries"才展开)——通用机制
  非 ABB 逐字;WAGO 论坛有客户端兼容反例(CODESYS 作客户端看不到数组,UaExpert 能看)。
  **实测项:UaExpert 对 VisuSlotState 整读一次**再定案。
- **许可**:官方付费 runtime 清单不含 OPC UA → 推断 V3 固件自带免费(排除法推断)。
- **通道决策不变**:Modbus TCP 仍为主推(§3b 配方全一手+pymodbus 生态);OPC UA 的比较优势=
  数组整节点原子读+订阅推送,若 Modbus 实测遇字序/线性化麻烦则切换,成本=Symbol Configuration
  勾选+asyncua 客户端,半天内可换道。

## 4. WebVisu 发布(T3.4 配方;⚠️0710 二枪修正)

步骤(逐字核实 `_visu_execute_webvisu.html`):Visualization Manager → Add Object →
**WebVisu Object** → 配起始画面+htm 文件名 → Show Used Visualizations 激活 → 编译下载启动
(**Web 服务器随应用自动启停**)→ 浏览器 `http://<PLC_IP>:8080/<htm名>`。

⚠️ **0710 重大修正(CODESYS Forge 官方回复,逐字)**:"if simulation is enabled on the device
this will disable all communication" —— **IDE Simulation 模式禁用设备全部通信,含 WebVisu 的
web 服务器**。连锁结论:①Simulation 下浏览器必然打不开,不是配置错;②Banner 在 Simulation
环境无处可眼见(IDE 预览不渲染+WebVisu 连不上),§2 的 V3 验证推迟;③WebVisu/Banner 的运行
验证三条路(待拍板):真机 AC500 / ABB 虚拟 PLC(AB2.9 是否带待查)/ CODESYS Control Win
软 PLC(target 不同,需评估工程移植成本);④**录屏主线=IDE 在线画面,不受影响**,报警可见性
由红色 Lamp+ALARM 状态文本承担(均非 Alarm 管理类元素,IDE 在线正常渲染)。
(通用 CODESYS 机制类推到 AB,ABB 专属论坛帖连接失败未能直读,置信度高但非 ABB 逐字。)

**V4 许可(0710 枪部分闭环,余一矛盾点)**:ABB 官方"licensed features"付费 runtime 清单
(help.plc.abb.com licensed_features_overview.html)**不含 WebVisu/TargetVisu,也不含 OPC UA**
→ 推断两者随 V3 固件自带免费(**排除法推断,非官方原话**)。矛盾点:老版 AB Release Notes 有
"Remote Target Visu…无许可 30 分钟停"字样,所指未明。终裁=真机实测或问 ABB;
AB GUI 内的 V4 验证(许可管理器搜一句原文)仍值得做,作交叉验证。
