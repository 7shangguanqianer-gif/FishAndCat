# 调研速览:这道题在学术界/工业界的成熟解法与工具链(2026-07-03)

> 结论先行:这是运筹学经典问题 **SLAP(Storage Location Assignment Problem,货位分配问题)** 在堆垛机立体库(AS/RS)场景的实例,套着"AI算法"外衣,落点是 PLC 工程实现。**每个环节都有成熟方法可直接采用**(语音赛纪律:先查成熟方案,不造轮子)。

## 1. 问题定性与经典方法

### 1.1 货位分配(SLAP)
- 中文文献主流做法:**多目标数学模型 + 遗传算法(GA)求解**。典型目标三件套:①作业效率(高频货放近出入口,最小化 Σf·t);②货架稳定性(重货放低层,最小化重心/倾覆力矩);③(可选)货物相关性聚集。参见[基于多种群遗传算法的自动化立体库货位优化(吉林大学学报)](http://xuebao.jlu.edu.cn/gxb/article/2018/1671-5497-48-5-1398.html)、[遗传算法求解立体仓库货位优化(CSDN)](https://blog.csdn.net/u011622208/article/details/70519823)、[基于自适应遗传算法的入库货位分配优化(中国储运)](https://www.chinachuyun.com/yuedu/yanjiu/157164770025515.html)、[现代仓储系统货物入库分配优化模型及算法研究](https://html.rhhz.net/GLJTKJ/20141224.htm)。
- 规则类基线(必做对照组):随机存放、就近存放、**ABC 分类分区**([ABC库存分类管理法](https://wiki.mbalib.com/wiki/ABC%E5%BA%93%E5%AD%98%E5%88%86%E7%B1%BB%E7%AE%A1%E7%90%86%E6%B3%95)、[双重ABC分类储位优化](https://zhuanlan.zhihu.com/p/266696682))、**COI 规则(Cube-per-Order Index,体积/周转率比小者放近处)**([COI仓库布局优化,IJPR](https://www.tandfonline.com/doi/abs/10.1080/00207549508930160)、[COI规则应用](https://scite.ai/reports/application-of-the-cube-per-order-index-DgbOwY))。
- 我们的架构判断:**在线规则评分(PLC 实时逐件分配)+ 离线/后台 GA(整批重排上界)** 双层。用 GA 最优值论证"在线评分规则的 gap 有多小",这比单独讲 GA 更有工程说服力。

### 1.2 堆垛机行程时间模型
- **切比雪夫(Chebyshev)模型是 AS/RS 标准**:堆垛机水平/垂直双电机同时运动,t = max(t_x, t_y)。参见 [AS/RS 行程时间模型与控制策略综述](https://www.researchgate.net/publication/282716003_Automated_Storage_and_Retrieval_Systems_A_Review_on_Travel_Time_Models_and_Control_Policies)、[分级存储下 AS/RS 行程时间计算](https://www.tandfonline.com/doi/full/10.1080/21693277.2020.1781703)。
- 曼哈顿距离用于"路径长度"统计,切比雪夫用于"执行时间"——两口径并列是懂行信号(见 canonical_assumptions C2)。
- 能耗+完成时间双目标的 AS/RS 启发式:[考虑订单完成时间与能耗的启发式(MBE 2024)](https://www.aimspress.com/article/doi/10.3934/mbe.2024006?viewType=HTML)——能耗目标direct对上 ABB 王余"绿色工程"致辞(姓名 2026-07-05 按官方开赛新闻订正,原误记"王玉")。

### 1.3 "AI"成色怎么保证(创新性10%)
- GA/多目标进化算法在这个赛题语境下就算 AI(官方新闻原文:"融合人工智能算法…设计多目标优化方案")。
- 可选加分不加险的点:①NSGA-II 产 Pareto 前沿图(比加权和好讲);②**扫描周期感知的分片 GA**——把 GA 迭代切片进 PLC 扫描周期非阻塞执行(有先例:[PLC Implementation of a Genetic Algorithm](https://www.researchgate.net/publication/2485013_PLC_Implementation_of_a_Genetic_Algorithm_for_Controller_Optimization)),这是 PLC 原生的工程创新,评委(ABB 工程师)会认;③强化学习只做离线对比实验,不上 PLC(语音赛 LoRA 教训:负结果也如实写,反而是工程叙事加分)。

## 2. PLC 侧关键事实

- **AC500 V3 = CODESYS V3.5 内核**,ST 语言支持结构体/数组/方法(面向对象),400 仓位评分全扫描是微秒级算术,在线决策毫无压力。
- **PLC 铁律:扫描周期内不可长阻塞**(看门狗超时=宕机)。GA 若上 PLC 必须状态机化:每周期跑 N 个个体/一代,跨周期收敛。参考 [PLC 扫描周期与看门狗设计](https://industrialmonitordirect.com/blogs/knowledgebase/implementing-watchdog-timer-circuits-in-plc-systems)。
- **Automation Builder Basic 许可证免费**,内置仿真模式可无硬件验证功能块([ABB Automation Builder 官方页](https://www.abb.com/global/en/areas/motion/digital-tools/automation-builder)、[下载页](https://www.abb.com/global/en/areas/motion/digital-tools/automation-builder/software-download)、[工程版说明](https://www.abb.com/global/en/areas/motion/digital-tools/automation-builder/engineering));完整虚拟调试要 Premium([论坛:Basic 许可证仿真模式](https://forum-controlsystems.abb.com/7051773/Simulation-Mode-for-automation-builder-v1-2-4-using-basic-license)),初赛用 Basic + CODESYS 仿真足够。
- 官方在线帮助:[help.plc.abb.com](https://help.plc.abb.com/)(赛题指定 AB 2.8.1 / AC500 V3 / CP600)。
- 决赛可视化 = CODESYS Visualization(表格/网格元素+ST 联动),官方教程链条齐全([CODESYS Visualization](https://us.codesys.com/products/visualization/)、[CoDeSys 可视化手册](https://crosscontrol.com/media/nyzg5ozx/codesys_visu_v23_e.pdf)、[ST 控制可视化切换](https://www.advantech.com/en-us/support/details/faq?id=1-1P3JXWD))。
