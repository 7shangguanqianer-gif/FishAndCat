# 《ST 编写标准》· 智储优控 PLC 侧(0707;评分细则4/6/3 交付物)

> 目的:统一 plc/ 目录 IEC 61131-3 结构化文本(ST)的命名、注释、接口、边界、同步规范,
> 支撑评分细则"结构化设计、代码及注释清晰""变量定义完整准确""功能块封装合理"。
> 依据:PLCopen Coding Guidelines v1.0(命名可自定义但须文档化,枪H 0707 已核对)+
> Beckhoff/CODESYS 前缀惯例(现有代码已遵循)+ 本项目 canonical_assumptions.md 口径。
> 地位:plc/ 新增或修改任何 POU 一律遵此;与本标准冲突的旧代码在 T2.2 重组时对齐。

## 1. 命名规范

### 1.1 变量前缀(Beckhoff/CODESYS 系,全项目统一)

| 前缀 | 类型 | 示例 |
|---|---|---|
| `x` | BOOL | `xExecute` `xDone` `xBusy` |
| `i` | INT/DINT | `iRule` `iBestX` `iGoodsCount` |
| `r` | REAL/LREAL | `rGoodW` `rBestScore` `rWMax` |
| `s` | STRING | `sLastFail` |
| `a` | ARRAY | `aSlots` `aGoods` `aRankIdx` |
| `st` | STRUCT 实例 | `stStats` |
| `e` | 枚举实例 | `eState` |
| `fb` | 功能块实例 | `fbInit` `fbAssign` |
| `GVL_` | 全局变量列表 | `GVL_Param` `GVL_WH` `GVL_Visu` `GVL_Data` |
| 大写全名 | 常量 | `COLS` `TIERS` `BASE_CAP` `MAX_LS_ROUNDS` |

> 依据:PLCopen 官方"前缀方案可自定义,但必须文档化"(枪H)——本表即本项目的文档化选型。
> 选 Beckhoff 系(BOOL=x)而非 PLCopen 示例系(BOOL=x 同,但 STRING=str 等略异),理由=
> 存量代码一致性(现有 1500 行全用此系)。**不得混用两系**。

**允许的例外**:短循环/坐标变量 `i j k m n`(计数)、`x y c t`(列/层坐标,语义命名优先于
类型前缀,与 canonical A1 的列 x/层 y 口径一致)可不加 `i` 前缀——这是 ST/工程惯例,
存量代码一致遵循,评审可接受。除此之外一律带前缀。

### 1.2 POU 命名(按类型+职责)

- `FC_<动词/名词>`:纯函数(FUNCTION),如 `FC_CalcScore` `FC_CheckFeasible`;
- `FB_<名词>`:功能块(FUNCTION_BLOCK,有状态),如 `FB_AssignAllGoods` `FB_Stats`;
- `PRG_<名词>`:程序(PROGRAM),如 `PRG_Main` `PRG_Test`;
- 语义与评分细则对齐(见 §6 追溯矩阵),评委可按名索引。

## 2. 头部注释块(每 POU 必备)

规范四件套(粒度=FB 头部块 + 关键行注释,0706 用户拍板):

```
(* ---------- <POU名>:<一句话功能> ----------
   功能:<做什么,对应 canonical 口径编号/评分细则号>
   输入:<IO 表——每个 VAR_INPUT 一行:名称 类型 含义 单位/范围>
   输出:<每个 VAR_OUTPUT 一行>
   边界:<越界/负输入/除零/满仓等如何处理,对应边界用例号>
   示例:<典型调用 + 已知期望值(便于对照 T19/T20 向量)>
   *)
```

- 生成文件(07_GVL_Data_generated.st)头部须标注"勿手改 + 生成脚本路径";
- 关键算法行(评分/优先级/tie-break/分片游标推进)必须行内注释解释"为什么这样写"。

## 3. FB vs FC vs PRG 选用判据(枪K/枪H 0707)

- **FC(FUNCTION)**:纯计算、无跨周期状态、相同输入→相同输出。本项目:`FC_CalcScore`
  `FC_CalcTravelTime` `FC_CheckFeasible` 等 12 个纯函数。
- **FB(FUNCTION_BLOCK)**:需跨扫描周期保持状态。两种接口子类:
  - **长任务型**(分片状态机):`Execute`/`xStart` 上升沿触发 + `xBusy`/`xDone` 汇报,
    内部 CASE 状态机每周期处理有限计算量(PLCopen Execute 模式,枪K)。本项目:
    `FB_AssignAllGoods` `FB_LocalSwapImprove`(200×400 计算分片,防看门狗)。
  - **计算型**(单周期):输入→输出即时,无握手。本项目:`FB_SelectSlot`(400 位单周期
    扫描)`FB_Stats` `FB_InitWarehouse`。
- **PRG(PROGRAM)**:任务入口,挂循环任务。本项目:`PRG_Main`(七态主状态机)`PRG_Test`。

## 4. 边界处理规范(枪I 0707;评分细则5)

- **归一分母保护**:任何除法前显式判零(`IF rMax <= 0.0 THEN rMax := 1.0`)——
  CODESYS REAL 除零返回 INF/NaN 且依目标系统而异(枪I 已核对),不可依赖。
- **数组越界**:CODESYS CheckBounds 默认静默钳位(不报错,枪I 官方双源核对)——
  **显式输入验证是必须而非可选**;坐标入参先 `IF x < 0 OR x >= COLS THEN` 拒绝。
- **错误接口**:长任务 FB 用 `xError:BOOL + iErrorId:INT`(枚举);交互层用状态文本
  `VisuStatusText`(ASCII 错误码,见 README §4b 清单)+ `VisuAlarm:BOOL` 红灯。
- **违规自检**:FB_Stats 末尾复查 ViolCnt(分配结果必须仍满足约束,恒 0,非 0 立即
  可见=程序缺陷护栏,评分细则"准确性")。

## 5. 双实现同步规范(ab_sync 铁律;评分细则"报告与实现互相背书")

改动传播链(任一环漏做则 T19/T20/T25 变红,这是设计出来的护栏):

1. 改 `sim/warehouse_sim.py` 的评分/优先级/权重/场景参数;
2. 重跑 `python sim/export_st_vectors.py` → 重生 `plc/07_GVL_Data_generated.st`;
3. 跑 `powershell -File tools/ab_scripting/ab_sync.ps1` → ScriptEngine 管线**自动**同步
   plc/*.st(含 07)进 AB 工程 + 编译 + 在线跑 PRG_Test(替代旧的手动粘贴,0707 落地);
4. 期望管线尾行 `iPassed=41`(=当前用例总数,随 T 增而增)、`iFailed=0`、编译 0 error;
5. 红了先想是不是忘了同步(护栏,非 bug)。

- 算法常量单源:`LAM_F/W/V`(Rank λ)、`MAX_LS_ROUNDS`由 07 生成,ST 侧 FC_CalcPriority/
  nMaxLsRounds **应引用**(T2.2 重组落地引用改造);当前值 0.50/0.35/0.15、5。
- STRING 字面量全 ASCII(中文触发 AB C0555 警告,W-005 已落地);注释中文无碍。

**"路径规划"口径诚实说明(0707 独立复核 F4)**:本题单巷道单堆垛机 + 切比雪夫双轴同动,
到任一库位走线唯一(先水平后垂直的 L 型),**无路径搜索/优化空间**——`FB_BuildVisuPath`

分配层**(AWRA-LS)。第一列保留官方细则原文,此处标注实现口径,避免"路径规划=路径搜索"被
评委误读为名不副实。
