# D-2 量化：静态权重 vs 自适应权重对头条数字的真实影响 — 2026-07-12（overnight R3）

> 目的：D-2 认定"PLC 跑静态默认权重、自适应权重是死代码"。本文用 sim **只切"权重"一个变量**（加减速 A4b 固定为 True，因 PLC 确已实现），干净量化 D-2 对报告头条数字的真实影响，给方案甲拍板提供数字底。
> 纯 python，不碰 AB；seed 固定（`headline.SEEDS=[2026,7,42,123,999]`），可复现。
> **一句话结论：D-2 对取货头条的影响很小（降幅 −1.6pp、取货 +0.35s、能耗≈不变、重货层高反而略安全、违规仍 0）——PLC 静态权重真跑仍是"降 60%/省能耗 67%"，诚实降级几乎不损故事。**

---

## 0. 三口径定义（关键：只有 P 才是 AC500 真实跑的）

| 口径 | accel(加减速) | adaptive(权重) | 对应什么 |
|---|---|---|---|
| **H 报告头条口径** | True | **True 自适应** | 报告 §1.1 现行头条数字（8.40s/62%…）|
| **P PLC 实际口径** | True | **False 静态默认** | **AC500 真跑**（A4b 已实现 + D-2 权重是静态 α1.0/β0.6/γ0.4/δ0.15）|
| L 对照口径（参考） | False | False | headline.py 的"匀速+固定"文献对照列 |

> D-2 = H 与 P 的差（只差权重）。R2 子代理误用 L（6.443s）当 PLC 复现值——L 同时改了加减速，隔离不干净；本文用 P 修正。

---

## 1. 结果（120件/skew/and，5 seeds，固定 seed）

| 指标 | H 报告口径 | **P PLC 实际** | L 对照(参考) |
|---|---|---|---|
| awra 期望取货 | **8.402 ± 0.477 s** | **8.757 ± 0.403 s** | 6.443 ± 0.356 s |
| seq 基线取货 | 22.116 s | 22.116 s | 20.251 s |
| 较 seq 降幅 | **62.01 %** | **60.41 %** | 68.18 % |
| 能耗较 seq 降幅 | **67.31 %** | **67.19 %** | 66.97 % |
| 批量 vs 在线 gap | 22.67 % | 20.28 % | 27.16 % |
| 重货平均层高 | 0.567 | **0.550** | 0.567 |
| 违规Σ(awra) | **0** | **0** | 0 |

**自检通过**：H 口径 awra 8.402s / 降 62.01% / 能耗 67.31% / 层高 0.567 与 `headline.py` 报告值 8.40/62.0/67.3/0.57 **逐位吻合** → 脚本口径正确、可信。

---

## 2. ★ D-2 影响（H → P，只切权重一个变量）

| 头条指标 | 报告 H | PLC 实际 P | 差 |
|---|---|---|---|
| awra 取货时间 | 8.402 s | 8.757 s | **+0.354 s（+4.22%，变慢）** |
| 较 seq 降幅 | 62.01 % | 60.41 % | **−1.60 pp** |
| 能耗降幅 | 67.31 % | 67.19 % | **−0.12 pp（≈不变）** |
| 重货平均层高 | 0.567 | 0.550 | **−0.017（更低=更安全）** |
| 违规Σ | 0 | 0 | **0（无安全回退）** |

**解读**：
- 自适应权重（D-2 的机制）对取货头条的贡献 **只有约 1.6pp / 0.35s**——远非"数量级"差异。
- 能耗降幅几乎不受影响（−0.12pp）。
- 重货层高在静态权重下**反而略低（0.550<0.567）= 略更安全**——因自适应权重更激进地压时间、略微牺牲重货层高；静态权重更保守。这是诚实、可讲的权衡，不是纯损失。
- **约束违规两口径都为 0**：静态权重不引入漏件/超限风险。

## 3. 对方案甲拍板的影响（重要）

**D-2 的诚实降级代价很低**，这实质降低了方案甲的叙事风险：

1. **PLC 真实数字仍然亮眼**：AC500 静态权重真跑 = **取货 8.76s / 降 60.4% / 能耗省 67.2% / 违规 0**——"降 60%+"的核心卖点完全成立，无需依赖未接入的自适应权重。
2. **推荐的诚实口径写法**（供方案甲采纳）：
   - 正文头条用 **P 口径（AC500 实现值）**：8.76s / 60.4% / 67.2%，标注"AC500 静态权重实现档"。
   - 把 **H 口径（8.40s/62%）降为"sim 设计余量：自适应权重可再提升约 1.6pp，为 PLC 后续扩展预留"**——从"主口径"变"设计上界/展望"，与 D-1 的 98.5% 处理同构。
   - 如此：数字全部诚实、PLC 名副其实、且自适应权重成为"未来可加分项"而非"现在的虚报"。
3. **与 D-1 的区别（勿混）**：本量化只覆盖**取货时间/降幅头条**（走 awra 策略，不经检测器路由）。**D-1 的"98.5% 检测器 attain"是另一条线**（检测器选档，holistic 81.6% vs lex 98.5%），本文未触及，仍需按 D-1 方案单独处理。

## 4. 边界与诚实声明

- 本量化仅 **sum 口径头条场景（120件/skew/and，5 seeds）**。其他场景/规模下 D-2 的差幅可能不同；若报告要对多场景声明"静态权重代价可控"，需扩跑场景矩阵（本轮未做，记为可选加深）。

- P 口径是"若把当前 sim 算法以静态权重在 sim 内重放"的结果，作为 AC500 真跑的**最佳 sim 代理**；严格的 AC500 实机数字需 PLC 端跑（本轮不碰 AB）。
