## 1. 变量命名(GVL_Visu 前缀族,存量惯例成文化)

| 前缀 | 语义 | 方向 | 示例 |
|---|---|---|---|
| `Visu*` | PLC→画面 显示量 | 出 | `VisuSlotColor` `VisuTotalTime` |
| `In*` | 画面→PLC 输入字段 | 入 | `InGoodId` `InQueryId` |
| `Cmd*` | 画面→PLC 按钮(BOOL,PRG 消费后自清) | 入 | `CmdAddGood` `CmdQuery` |
| `Sel*` | 画面→PLC 选择器(INT 档位) | 入 | `SelStrategy` |
| `Qry*` | 查询页输出组 | 出 | `QryFound` `QryX` |
| `aRow*` | 记录页 10 行窗口并行数组 | 出 | `aRowId` `aRowExecT` |

规则:①画面绑定只许绑 GVL_Visu/GVL_Param(不绑 FB 内部变量)②按钮一律"上升沿消费+
PRG 清零"(存量 CmdAddGood 模式)③新增变量必须带行注释(含单位/编码表)。

## 2. 颜色语义(单源 FC_StateToColor,禁画面写死状态色)

状态编码(存量 0-5)+ 块3 新增 6:
`0 空闲(浅灰) 1 预占用(深灰) 2 已占用(绿) 3 推荐(黄) 4 路径(蓝) 5 报警(红) 6 查询高亮(青)`
- 热力方向遵行业惯例:暖=忙/热,冷=闲(盲区补检 B 节);**判读话术**:健康布局=热点聚
  I/O 口向外衰减,劣布局=热点散落——对比演示的讲解主线。
- 对比模式(CmdCompareMode)下显示快照数据,色表不变(同一 FC_StateToColor)。

## 3. 布局网格(Swisslog 分屏范式)

```
┌──────────────────────────┬───────────────┐
│  左 60%:20×20 热力网格   │ 右 40%:      │
│  (VisuSlotColor 参数化    │  输入区五字段  │
│   子可视化)+堆垛机图元    │  推荐/状态卡   │
│                          │  统计小卡      │
├──────────────────────────┴───────────────┤
│  底:Alarm Banner 报警条 + 状态文本 + 徽章角标 │
└──────────────────────────────────────────┘
```
P2 统计记录页/P3 查询页同构:左=主内容(表/查询结果),右=操作区,底=常驻条。
字号三级:标题 20+/正文 14/数值突出 16 加粗。**一屏一焦点**(HUD 原则)。

## 4. 交互反馈原则(游戏 HUD,方向8)

- 任何按钮按下→可见反馈 ≤1 个扫描周期(状态文本/颜色/数值至少一样变);
- 输入超范围→NumPad Min/Max 拒绝式(首选)+ 状态文本说明原因(存量 ERR_* 模式);
- 危险操作(Reset)按钮独立放置远离高频按钮;
- 演示流程内每一步的"评委该看哪"在操作卡里显式标注(视线引导)。

## 5. 输入控件配方(一手,施工依据 §1 摘录)

- 数值:Rectangle + Input configuration→**OnMouseClick**→Write a Variable→
  `VisuDialogs.Numpad`,Min/Max=拒绝式;
- 字符串:同上 `VisuDialogs.Keypad`;
- 名称下拉+手输:`Combo Box, Array` 绑 `astrNames`(写 INT 下标)→ ST 同步 stName
  → 同变量叠加 Keypad 文本框;
- 报警:Alarm Configuration→Class→Group(Digital 绑 `GVL_Visu.VisuAlarm`=TRUE)→
  Alarm Banner 元素(勾组/类,Filter=Newest,列 State/Message/Timestamp)。

## 6. WebVisu 约束(先验后建)

- 400 格阵列必须用**参数化子可视化+数组下标**批量实例(禁手摆 400 元素);
- 开工验证 V1-V5(施工依据 §5)未过前,不使用 Path3D/Depictor/Trend 于关键路径;
- 3D 保底=2.5D 伪 3D(位图+堆垛机图元绑坐标),真 3D 通过 V2 验证才启用。
