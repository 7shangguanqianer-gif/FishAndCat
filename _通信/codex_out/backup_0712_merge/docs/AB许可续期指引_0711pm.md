## 0. 一句话结论

**最可能的情况：你在装机/建项目时激活了 Automation Builder 的"30 天免费试用许可"（解锁 Standard+Premium 全功能），而不是购买的订阅或免费 Basic 版**——因为 Basic 免费版本身**没有到期日**，会弹出倒计时的只有试用版或付费订阅版，而你的项目文档最早日期在 0706-0707，与"30 天前后激活试用、8 月初到期"的时间线吻合。但这是**推断**，不是确认；真正的确认方法见第 1 节，你必须先做这一步，才能决定走哪条续期路径。

## 1. 第一步：先确认许可类型和到期日（不要跳过，不要先猜）

**不要直接开 Automation Builder**（避免触发不必要的激活/弹窗交互）。改为：

1. 开始菜单 → CodeMeter → **CodeMeter Control Center**（你机器上已装）
2. 主界面会列出本机安装的许可证"容器"（CmContainer / ticket）
3. 点 **[WebAdmin]** 按钮，会在浏览器打开更详细的 CodeMeter WebAdmin 页面
4. WebAdmin 里进 **Content / Container → All Container**，找到含 "Automation Builder" 字样的条目
   - 注意：Automation Builder 的一个版本会对应**多个 product code**，需要认准条目名称里写明版本的那个（例如 "Automation Builder Standard"／"…Premium"／"…Trial"）
   - 记录：**许可版本**（Basic/Standard/Premium/Trial）、**精确到期日期**、**容器类型**（PC-bound 单机软容器 / USB 加密狗 dongle / 网络许可 server）
5. 之后如果正常开一次 Automation Builder，标题栏也会显示当前生效的许可证类型，可与上面交叉核对

> 关于机器上的 "**Automation License Manager**"：本次调研没有查到 ABB 官方文档把它作为 Automation Builder（PLC 工程）许可体系的组成部分——它更可能是 ABB Drives 相关工具（如 Drive Composer/DriveWare 的许可管理），和 Automation Builder 走的 CodeMeter 体系不是一回事。**不要假设它管理的就是 AB 的许可**，开机后点开看它的标题/关于页确认到底是什么，再判断是否相关。这属于弱证据推断，未在官方文档中直接证实。

## 2. 续期路径（按可能性从高到低排序）

### 路径 A（最可能）：30 天 Trial 试用许可到期

官方确认：Trial 试用许可有效期 30 天，解锁 Standard+Premium 全部功能；试用到期后，若要继续使用 Standard/Premium 功能，"需要向 ABB 订购新许可或升级许可"（原文：*"you will need to order a new or upgrade license from ABB"*）。免费 Basic 版是独立的许可类型，不会自动获得——需要在许可向导里单独选择/激活。

**先做一个判断题**：你项目后续阶段是否用到了超出 Basic 免费版范围的功能？
- Basic 版**不含**：Fieldbus 现场总线连接、Drive Composer pro、以及 Standard/Premium 的运动控制(Motion)/机器人(Robotics)集成工程、安全 PLC(Safety PLC)、C/C++编程、E-CAD 接口等高级功能
- 如果你的工程只是常规 IEC 61131-3 编程（ST/梯形图等），**不涉及**上述项，Basic 免费版大概率够用
- 如果你不确定，这是"需要你自己先确认"的关键项之一（见第 3 节）

**分支 A1：Basic 免费版够用 →**
1. 让试用自然到期，无需抢在 8/2 前做任何事
2. 到期后如果 Automation Builder 弹出"试用已到期"的提示反复跳出（即便你后面选了 Basic），参考官方页面 [Removing trial license to remove expiring message](https://help.plc.abb.com/removing_expiring_message.html) 删除旧的试用许可记录以消除提示（删除操作不可逆，注意该页提示：如果你删除的许可本身还在生效，会永久丢失，操作前确认状态）
3. **务必在到期前用第 1 节的方法先验证一次**"目前工程确实没用到 Basic 不支持的功能"，不要到 8/2 当天才发现用了 Fieldbus/Motion 之类导致工程打不开/降级

**分支 A2：需要继续用 Standard/Premium 功能 →**
1. 优先渠道（并行做，不要等一个渠道有结果再试下一个）：
   - **联系 2026 ABB杯组委会**：赛事由 ABB 冠名，很多厂商冠名的学生赛事会为选手提供软件许可支持或试用延期。官网 [2026 ABB杯智能技术创新大赛](https://new.abb.com/cn/innovation/2026-abb-cup-innovation-contest) ——**本次调研两次尝试抓取该页面均超时失败，没能验证是否真的写明许可支持条款，这是本次调研的一个负结果，你需要自己打开这个链接或找赛事群/公告确认**，这可能是最快最省钱的路径，优先级应该最高
   - **联系 ABB 官方销售渠道**：通过 [new.abb.com/cn](https://new.abb.com/cn/en) 官网联系表单，或 Automation Builder 产品页找 "Contact us"，说明需要一个 Automation Builder Standard 或 Premium 的**单机版(single PC license)**许可（学生个人用途，不需要网络版）
   - 官方经销商查找入口：[ABB经销商专区 mall.abb.com.cn/distributor.html](https://mall.abb.com.cn/distributor.html)（这是官方页面本身，第三方经销商资质以此页确认为准，本次调研中搜到的具体经销商名称属于非官方信息，仅供参考不作为唯一渠道）
   - 本次调研**没有找到** ABB Automation Builder 面向高校学生的专门"教育版/学生许可"项目（ABB University 有面向员工/合作伙伴的培训和 eLearning，但那是培训资源，不等同于给你一份可续期的产品许可）——如果 ABB杯组委会也没有相关支持，大概率只能走正常商业采购/申请免费延期两条路
2. 拿到激活码后，在 Automation Builder 许可向导里：启用"单台电脑许可证" → 启用"在线激活" → 输入激活码 → 联网激活，官方文档描述这一步在线激活完成得很快（几分钟量级），**真正不确定、需要预留时间的是"拿到激活码"这个商务/审批环节**，不是技术操作本身
3. 如果到时候这台电脑没有网络，走离线激活：生成许可证申请文件 → 用另一台能上网的电脑，把申请文件上传到 ABB 官方 CodeMeter Depot：`http://lc.codemeter.com/32838/depot/index.php` → 下载返回的 `.WibuCmRaU` 授权文件 → 拷回本机，在 Automation Builder/CodeMeter 里导入 → 再把生成的回执文件传回同一个 Depot 网址完成闭环。详见官方页面：[Activation without internet connection](https://help.plc.abb.com/activation_without_internet_connection.html)、[Offline activation](https://help.plc.abb.com/offline_activation.html)
4. 如果新许可要装到同一台电脑上、且旧的试用许可还占着，可能需要先"退还/转移"旧许可释放容器：打开 Automation Builder 菜单"帮助 → Return of Automation Builder license"，或直接访问 `http://lc.codemeter.com/32838/depot/index.php`，在 Ticket id 里填入（旧）激活码 → 选 "Re-Host Licenses"，系统会自动判断走在线还是离线转移流程。详见：[Returning an Automation Builder license](https://help.plc.abb.com/returning_automation_builder_license.html)

### 路径 B（较不可能，但需排除）：付费 Standard/Premium 订阅到期

如果第 1 节里 CodeMeter WebAdmin 显示这是一个 subscription（订阅/维护到期），而非 Trial：官方文档提到"产品订阅的许可更新是自动完成的"（原文：*"License updates for product subscriptions are done automatically"*），如果这台电脑能定期联网，系统可能会自动续期，不需要你手动做什么。但既然登录弹窗主动提示了倒计时，**不要단纯假设它会自动完成**——建议按第 3 节列的方式，直接联系当初这份许可的采购方/ABB销售确认续费流程和账单，双重确认，不要赌自动续期一定生效。

### 路径 C（边缘情况）：网络许可/加密狗借用到期

如果第 1 节确认容器类型是"网络许可(server)"或"借用(borrowed)"，说明许可是从学校机房的许可服务器分下来的，续期需要找机房管理员/设备老师协调，不是你个人能操作的范围，直接去问负责机房设备的老师即可，不用走 ABB 官方购买流程。
