# 工业 HMI 交互惯例调研(0706 定向枪,反哺 v3 设计稿与批次 C)

> 来源:sonnet 定向枪,ISA-101/ISA-18.2/厂商指南/CODESYS 官方帮助;URL 全留。
> 用途:①v3 设计稿依据 ②批次 C 操作卡的控件选型 ③报告里"遵循工业标准设计"一句话的证据链。

## 一、六条核心事实(每条附关键来源)

1. **ISA-101(ANSI/ISA-101.01-2015)**:背景必须浅~中灰,**明确禁止黑底**("never black; glare causes fatigue");正常状态视觉静默(灰),**颜色只用于异常**("If everything is coloured, nothing stands out");禁止渐变;四级画面层级(L1 全厂概览→L2 单元→L3 设备细节→L4 诊断)。
   https://blog.ansi.org/ansi/ansi-isa-101-01-2015-hmi-for-process-automation/ ; https://hmilibrary.com/standards/isa-101
2. **参数调节控件工业主流 = 数值输入框+NumPad 弹窗+写前确认**(显示旧值→新值,Confirm 才写 PLC);滑条在工业 HMI 少作主输入(精度差/误触);+/- 步进适合离散档位。
   https://plcprogramming.io/blog/hmi-design-best-practices-complete-guide ; ASM/Honeywell 白皮书(见调研原文)
3. **ISA-18.2 报警**:优先级色 P1 红/P2 橙/P3 黄/P4 蓝白;**只有"活动-未确认"状态才闪烁**(约 4 次/秒),确认后稳定色;颜色不得是唯一区分手段(色盲 8-12% 男性)。
   https://www.mikrodev.com/alarm-management-isa-18-2-scada-alarms-event-management-priority-and-operator-effectiveness/
4. **页眉标准件**:画面标题+实时时间戳+操作员标识+模式指示(Auto/Manual)+**报警汇总条(全画面常驻,不得被遮挡)**。WinCC 模板固定四件(Operator/DateTime/Language/Diagnostics);PlantPAx 报警条边框色=最高级活动报警色。
   PROCES-RM200: https://literature.rockwellautomation.com/idc/groups/literature/documents/rm/proces-rm200_-en-p.pdf
5. **CODESYS 原生支持全套**:VisuDialogs.Numpad(带上下限)/Open Dialog 确认框/Alarm Banner/Alarm Table(含确认按钮向导)/Alarm Class(每状态独立配色;**无内置闪烁,需颜色变量绑定自实现**)。
   https://content.helpme-codesys.com/en/CODESYS%20Visualization/_visu_configure_text_input_with_virtual_keyboard.html ;
   https://content.helpme-codesys.com/en/CODESYS%20Visualization/_visu_creating_an_alarm_banner.html
6. **红黑配色风险(实锤)**:黑底被标准与厂商指南一致反对;大面积红→报警脱敏("if using red for P1 alarms, do not use red to represent a running pump");黑底红字是色盲最常见失效通道,低亮度屏可读性差。
   https://www.realpars.com/blog/hmi-colors ; https://centerforoperatorperformance.org/projects/background-color-hmi

## 二、对本项目的裁决:红黑品牌框 + ISA-101 灰工作区(v3 方案)

用户拍板"A 改红黑变体"→ 与事实 1/6 冲突。化解:**红黑收进品牌区**(顶栏黑底+ABB 红字标/红色点缀线),**工作区(网格+右面板)按标准中灰**,红色专属报警格(深红 #B71C1C+黑粗边,与 Q2 拍板一致)。收益:品牌记忆点保留 + 报告可写"遵循 ISA-101/ISA-18.2 设计"变成加分项。v3 设计稿并排给"全红黑"对照版,用户看稿终拍。

## 三、右面板五条改造(进 v3 与批次 C 操作卡)

1. 面板底色黑→中灰(#3C3C3C~#505050),文字浅灰/白【依据 1/6】;
2. 权重参数=显示当前值→点击弹 Numpad(带上下限)→确认框显示"旧值→新值"→Confirm 写入【依据 2/5】;
3. 红色只留异常:正常占用格改中性色(蓝/灰),红=故障报警专属【依据 1/6;与既有 FC_StateToColor 六色对表】;
4. 面板顶部加 Alarm Banner(CODESYS 原生件,显示最高优先级活动报警,常驻)【依据 4/5】;
5. KPI 数字按异常才着色(正常灰白/告警黄/报警红),页眉补时间戳+模式指示(SIM/RUN)【依据 3/4】。
