## 267 与 400 的最终解释

1. 20×20 共 400 格，官方 `(列+层)%3==0` 预占 133 格，算法可用上限为 267 格。
2. 因此 canonical 可以在条件满足时达到静态 `400/400`：133 PRE + 267 管理货物；不是“只有 267 格”。
3. `400/400` 不是现有模型的持续运营能力。当前 mixed cycle 先入库后出库，满库后没有空位接收下一件；持续满库作业需另建“先出后入”或缓冲区模型。
4. 现有 S3 是 120 件库存、100 个 paired IN/OUT 周期、单 seed 代表回放，总占用固定约 253/400，不是满放证据。
5. 无预占 `0→400` 可以另做 Python sim 容量反事实，但必须新建 `none` 规则、独立 schema/测试/入口，并常驻“sim-only / 非题面口径”；不得混入 canonical、PLC、FIO 或 KPI。

## 三名 5.6-sol 独立审查新增发现

### 阻断与严重项

1. **fill 不能复用现有 trace**：现有引擎是 paired IN/OUT，库存净值不增长；必须新增 `fill-only` runner 和 `s3-fill-comparison-v1`。
2. **267 是容量上限，不是通用能力**：满放可行取决于货物与策略；SCORE/AUTO 在 267 密度在线权重入口存在 fail-closed 风险，必须先冻结 profile/seed/货序/权重/归一化并正式证明 267/267、0 fail、0 viol。
3. **AWRA-LS 信息集不同**：它是离线批量 Rank→Assign→局部搜索，只能作为离线批量参考；在线主对比必须使用同信息集策略。
4. **失败域吞吐可能虚高**：`mixed_ops.py` 的固定计划操作数会在入库失败时继续进入吞吐分子；页面若有 fail/viol 或完成数不齐，吞吐必须 `N/A`，或以真实成功操作数重新派生。
5. **LAP 超容量守门缺陷**：`n>m` 时 `linear_sum_assignment` 可返回仅 m 条边，而现有函数未校验完整分配；修复前不得把 LAP 当 401 件或超容量 oracle。
6. **当前 S3 没有路径、kg·m 能耗、吞吐字段**：`travel_dual_s` 是时间，不是米数；三维轨迹也不是路径 KPI。先扩展 trace 定义/单位再展示。
7. **“完美摆放比例”无正式定义**：改用入库成功率、违规数、受限 VBS attainment 或同实例真实 LAP gap；98.5% 不能称理论上限。
8. **Python/ST accel=true 尚非同源评分**：ST 的 `FC_TMax` 与 Python 动态分母/装卸边界存在差异；S3 只能标 sim，不得借画面宣称控制器同算法一致。

## 新数据合同

- 顶层：`shared_input`（容量、scenario、seed、cargo、到达/噪声/故障、source/config hash）。
- lanes：算法 ID、`online/offline_batch` 信息集、267 个 events、lane result hash。
- event：gid、decision、selected slot、candidate audit ref、SIM timestamps、phases、owner、occupancy before/after、faults。
- slot 状态：`PREOCCUPIED / EMPTY / CLAIMED / OCCUPIED`；只有 `RACK_STORED` 才 +1。
- fill phase：`QUEUE→DECISION_LOCKED→CARRIER_IN→STORE_HANDLE→RACK_STORED→EMPTY_RETURN→COMPLETE`；不得夹入出库或无物理依据的 `SCANNED/EXIT_MASK/CONSUMED`。
- checkpoints：0、67、134、201、241、267 件；快跳 reducer 必须等价于从 event 0 重放。

## 对比指标合同

- 结果/安全：成功、失败、拒收、违规、终止原因、双计数。
- 时间/距离：双程时间 s、双程路径 m、makespan s、响应 P50/P95 s。
- 布局：期望取货 s、hot20 s、heavy20 层、重心 m、利用率。
- 资源：垂直势能代理 kg·m、忙时、队列峰值、故障与停机；kg·m 不是 kWh。
- 差值：绝对/%；分母 0、失败域或样本不齐时 N/A。
- S1 统计与 S3 回放分层；单 seed 动画不替代多 seed 排名/CI。
