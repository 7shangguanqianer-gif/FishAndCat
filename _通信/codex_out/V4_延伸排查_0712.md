## 总结论

1. **oracle_gap 可复现性完全通过**：3个CSV、summary和fig13共5个产物与现有文件逐字节相同，SHA256全部一致；D-1数字稳定可复现。
2. **8.40s/62.0%/67.3%/0.57 全部是 `accel=True + adaptive=True` 的 sim H口径，不是当前 PLC端到端实测。**
3. 独立量化复现了“只去掉自适应权重、仍手动开启加减速”的结果：8.757s/60.41%/67.19%/重货层0.550；D-2对主头条影响较小。
4. **新发现：不能把上述静态+加减速组合直接叫“PLC实际口径”。** 当前 PLC源码 `xUseAccel` 默认 `FALSE`，生产代码无置 TRUE 路径，三页HMI施工文档也未提供该开关；T25端到端测试显式强制 `FALSE`。除非存在未落盘的人工在线设值，PLC默认路径是“匀速+静态权重”。
5. **230次/h吞吐与年化kWh/CO2属于 sim模型能力，PLC没有对应运行状态机/年化换算。** PLC只有双命令时间公式和能耗代理统计，不能称已在AC500验证这些头条。
6. `59/0` 证明 ST子功能和静态匀速黄金向量正确，不证明 H口径性能KPI在目标控制器上复现；报告中“仿真结论已在目标控制器上闭环验证”的裸句必须收窄。

## 三、新发现 N-1：PLC加减速实现了，但默认未启用

### 证据

- `plc/02_GVL.st:28-38`：`GVL_Param` 是普通 `VAR_GLOBAL`，`xUseAccel : BOOL := FALSE`；不是 RETAIN/PERSISTENT。
- `plc/03_Functions.st:131` 与 `:373`：功能代码确实支持 TRUE 时走梯形加减速，故“功能已实现”成立。
- 全部生产 ST 赋值只见 `FB_ParamGuard` 把快照值写回；没有任何启动时 `xUseAccel:=TRUE`。
- `plc/06_PRG_Test.st:133-143` 的 T22 仅暂时置TRUE验证轴时间，随后恢复FALSE。
- `plc/06_PRG_Test.st:161-166` 的 T25 AWRA端到端护栏明确写“sum官方口径，匀速”，并强制 `xUseAccel:=FALSE`。
- `2_你要操作/AB可视化搭建操作卡.md` 与 `docs/画面三页制施工蓝图_0711.md` 只提供α/β/γ/δ输入，未发现 xUseAccel 控件或演示步骤。

### 判定

**高优先级口径错配。** 当前文字常把“PLC已实现 A4b”进一步推成“PLC当前运行口径=accel=True”；前半句是代码能力，后半句没有仓库证据。由于红线禁止打开AB，本轮不能排除用户曾在在线监视页临时手动置TRUE；但该值无持久化/文档化路径，不能作为可复现的默认配置。

对 `docs/D2量化_0712.md:9-17` 的独立判定：数值计算正确，但把 P列命名为“AC500实际/真跑”过强。建议改为“PLC可实现的静态+加减速sim代理（需先明确把 xUseAccel置TRUE）”。

## 四、其他头条数字的 sim↔PLC 边界

### 1. 吞吐230次/h

- `sim/mixed_ops.py:3-15` 定义100周期、每周期1存+1取的混合作业仿真。
- `:113-130` 用 `--accel --adaptive` 形成 H口径；230.44次/h由该仿真输出。
- PLC仓内只有 `FC_CalcDualCycleTime` 公式（`plc/03_Functions.st:146-160`）和 T23单个公式用例（`06_PRG_Test.st:146-150`）；没有出库请求、混合流循环或吞吐状态机。

独立重算：H=230.44次/h；加减速+静态权重=226.96次/h。两者都只是 sim。**230不能称PLC实测/PLC运行吞吐。**

### 2. 年省电266.5kWh/降48.9%、电费与CO2

- `sim/energy_report.py` 使用η、μ、载货台自重、500次/日、300日/年、电价、排放因子等假设，将仿真作业流换算成年值。
- PLC只计算 `Σ重量×高度` 的 kg·m能耗代理（`plc/04_FB_Warehouse.st:411-444`）；没有kWh、年化、电费或CO2逻辑。

这些数字可作为“sim工程情景估算”，不可说“AC500测得/PLC算出”。

### 3. 59/0与“仿真结论在控制器闭环验证”

- T25黄金向量由 `sim/export_st_vectors.py:28` 的静态权重生成；`06_PRG_Test.st:165-166` 还锁为匀速。
- 59/0能证明当前ST实现、边界函数、静态匀速AWRA向量等测试通过；不能证明自适应权重、lex/speed检测器、5-seed H KPI、混合流吞吐或年化能耗在PLC复现。
- 因此 `docs/报告草稿_B1B2B10_0711pm.md:140-149` 的“建议直接点出‘仿真结论已在目标控制器上闭环验证’”属于叙事过度。应改成“核心ST逻辑与静态匀速黄金向量已在AB仿真通过59/0”，并另列未验证项。

### 4. 可视化设计稿中的8.40s

`2_你要操作/可视化屏幕设计稿.html:58` 静态写入8.40s。它是设计稿KPI示例，不是PLC实时变量证据；若截图进入材料，必须标“sim H口径示例值”，否则会形成新的画面—实现错配。

## 五、可复现性/数据治理附带问题

1. `sim/headline.py:6` docstring仍写场景规则 `and`，实际 `run_five():43,45` 已硬编码 `sum`；数字本身正确，但复现说明过时。
2. `sim/mixed_ops.py:160-168` 的CSV只记录 `motion=trapezoid/uniform`，不记录 `adaptive`。同一路径若不带 `--adaptive` 重跑，会覆盖成不同权重口径而文件无法自证。建议新增 `adaptive`/`weight_mode` 列。
3. `tools/doc_check.py`不检查这些语义口径，必须保留专门的“sim数据 vs PLC实现”审计。
4. A5的“单次调用1周期”不能推出7策略必超10ms；真机CPU裕量仍未测，见V3。
