## 六个最重要的新结论

1. **8.757/60.4 代理还需降级**  
   Python 加减速评分分母随 ACCEL 变为 39.6667 s；ST FC_TMax 固定用匀速 38 s。现有测试只测加减速时间、不测加减速评分。更接近当前 ST 数学式的 Python 只读代理为 8.794 s / 降 60.24% / 能耗降 67.07%，仍不是 AB/AC500 实跑。

2. **权限守卫会“值回滚、布局不回滚”**  
   PRG_Main 先执行分配/局搜，后执行 fbGuard。越权参数可先选位并留下永久副作用，再被回滚。T55 只测值，没有测消费者副作用。

3. **演示批重复载入可数组越界**  
   FC_LoadDemoGoods 每次无条件追加 20 件，无 MAX_GOODS 检查；空表第 21 次会写 aGoods[401]。

4. **H 图表源混口径**  
   PPT “H 主结果”页嵌 fig1，但 fig1 是 ACCEL=False+静态权重；fig1/fig5 图内仍写 and，代码实际 rule=sum。改完 PPT 文本直接重生仍会把错图重新注入。

5. **run_all 不是全量再生**  
   它不跑 oracle 两测试、不跑多数专项实验、不生成报告/PPT，却把 out/figs 全部旧文件打印成“产物”。build_report 缺数据仍保存并 exit 0。

6. **孤儿 DOCX 绕过所有联动**  
   项目全景说明书.docx 无同名 md、无当前生成器覆盖、doc_check 不检查，仍保留 7.42/66.2/74.3、356/152、23/18 tests 等整套 0704 快照。
