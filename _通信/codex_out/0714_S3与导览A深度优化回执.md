## 2. S3 深度优化

### 2.1 解决“货柜间隔与阴影同色”

1. 将统一 emissive 改为按部件分配：内板保持低自发光、前缘提高亮度、投影保持暗部，形成“亮前缘—中性内板—暗投影”三角色。
2. 400 个内板不再接收堆叠阴影，避免格内灰阶被阴影污染；货架主体和地面仍接收阴影。
3. 在不改变 1 m 单元与 20×20 几何的前提下增加 5×5 宏网格，帮助远景先读出结构分区，再读单格。
4. 主方向光仍为 0.35、headlight 仍为 0.25；只增加 PCFSoftShadow 的 `normalBias=.018` 和 `radius=4`，消除硬边与自阴影脏线。

对应落点：S3 `M` 材质分工约 155–160 行；内板与宏网格约 173–183 行；阴影参数 247 行。

### 2.2 机构与信息表达

- 增加列号与层号双向标尺，远景也能定位目标格。
- I/O 区补齐输送机侧梁与滚筒，三只排队货箱继续统一引用 `IO` 锚点。
- 黄金单桅杆不变；补充移动滑座、顶部横梁、驱动箱、导向鞋与载货台框架。所有移动附件与桅杆共用 world X。
- 去程/回程仍由 `axisPos` 逐时刻采样的分段线性曲线生成；增加方向锥并按当前相位提升对应轨迹透明度，没有改成贝塞尔或样条。
- 增加四相位时间轨、X/Z/T 轴状态 HUD、目标格轻微脉冲、货物重量色图例和速度图 X/Z 图例。

### 2.3 黄金不变量确认

- 几何：`N=20`、`Z0=-0.5`（S3 106 行）。
- 运动：`VX=2.0`、`VZ=0.5`、`AX=0.5`、`AZ=0.3`、两端驻留 4 s；`axisT/axisPos/axisV` 原公式不变（S3 107、115–139 行；formal 45–46 行）。
- 路径：仍使用 `PiecewiseLinearCurve`（S3 213 行），没有引入 Catmull/Bezier。
- 相机：`FOCAL=(10,.5,8)`、`EL=18°`、`R=42`、Z-up（S3 251 行；formal 418–427 行）。
- 灯：方向光 0.35 + headlight 0.25，两灯强度不变（S3 246–249 行；formal 196–198 行）。

## 3. 导览 A 重构

### 3.1 信息架构

旧版是一张连续长页，顶部导航、材料卡和模块卡都重复承担“跳转/打开”动作。新版改为固定视口的评委阅读工作台：

- 左侧：7 个有限章节与进度；支持点击、↑↓、←→、Home、End。
- 中间：一次只显示一个章节；顶部只保留一个“下一章”主动作。
- 右侧：当前章节的结论、口径、来源/命令和边界；证据随章节切换。
- 顶部：5 个关键指标常驻，避免切章后失去全局锚。
- 首页只提供三个互斥评审意图：先看结论、先看工程、直接查证。

章节为：首读总览 / 核心结论 / 模型与算法 / 控制与 HMI / 技术预演 / 展示模块 / 材料与复现。

### 3.2 去重与渐进披露

- 6 个材料条目只负责选择；“打开所选 / 复制完整路径”集中到右侧唯一动作区。
- 浏览器检查：材料卡内部重复动作数为 0；上下文动作数为 2。
- 10 个模块是选择器而非伪入口；统一标注“○ 建设中”，点击只更新一处详情和证据栏。
- 模块 09 保持准确名称 `数字孪生(C6 自研)`。

### 3.3 成熟交互依据

- 使用 W3C ARIA Authoring Practices 的 tabs 语义：`tablist/tab/tabpanel`、`aria-selected`、`aria-controls` 与键盘切换。
- 采用渐进披露：初始只显示当前任务需要的信息，把来源和边界放入随章节变化的证据栏。
- 本地 `FactoryIO三课过程教练_20260711.html` 仅借鉴了“有限 hash 路由 + 固定侧栏 + 面包屑 + 进度 + 上下文动作”，没有复制其教学语气和大量按钮。
