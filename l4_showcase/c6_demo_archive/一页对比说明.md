## 两栈这次各自证明了什么

- **PyVista**:VTK 硬件 Z-buffer 全程零遮挡穿帮(0706 mplot3d 的老大难);原生出 mp4,
  就是 B12 正片(90-120s)的生产形态;中文 KPI 浮层(微软雅黑)直接可用。
- **Three.js**:同一场景在浏览器里 60fps 自由拖拽;`canvas.captureStream+MediaRecorder`
  录制按钮实测可导 webm(官方机制);若后期想要"评委自己转着看"的交互交付,这条路已铺通。

## 诚实声明(口径纪律)

1. KPI「平均单程 9.52s」= 120 件单程行程时间(CSV `travel_time_1way_s` 列)的简单均值,
   **不是**报告头条的 8.17±0.67s(那是 30-seed 频次加权期望取货时间)。demo 仅演示
   数据管线,正片会换成与报告一致的口径。
2. 两栈均为 **sim 数据回放**,画面标注 (sim);无报警情节故全片无红色(红=报警专属)。
3. Three.js 页在我的无头验证环境里 RAF 被节流(逐步 API 验证了 120/120 全部逻辑),
   动画流畅度请以你本机浏览器为准——这正是要你过目的原因。
4. 色板:蓝三档=轻/中/重(重货深蓝沉底叙事),琥珀=正在放置件+I/O 口,服从 v3 纪律。
