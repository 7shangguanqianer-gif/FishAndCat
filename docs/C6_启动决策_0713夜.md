## 1. 为什么现在是 C6（战略定位）
- F5 实测：AC500 仿真做 Modbus 主站，运行期连接建不出（pFC 指针 NULL，与 F4 OPC UA 同构的仿真边界）。
- 连带结论：**整条在线门线 C4 单任务 / C5 三任务在线门被同一仿真边界阻塞**（需真实 AC500 硬件），F6 回补依赖 C5 绿→也阻塞。
- **C6 是纯软件交付**：用我们自己的 3D 视图回放**同源事件流**，不依赖 Factory I/O 的商业 3D / 授权 / 硬件——正好是"硬件阻塞"的软件答案，叙事上直接支撑"数字孪生"卖点。用户已授权"C6 提前"。

## 2. 本轮已建（数据层，纯软件、可单测、不碰绿码）
C 系列离线门早已产出结构化事件：`event_log.TaskEvent`（含 operation/status/phase/
physical_source/target 格位/fault_code/timestamp）+ `JsonlEventLog` 追加写 JSONL +
`TaskOrchestrator`（C2 编排门）执行任务即发事件。**这就是"同源"数据源**（驱动真实系统与喂视图的是同一套事件契约）。

本轮新增（`l2_factoryio/`）：
- **`c6_replay_state.py`** — 同源事件 JSONL → 逐帧仓库状态：每帧含堆垛机当前服务格、
  **占用格集合（确定性重建：DONE 时 INBOUND 占位 / OUTBOUND 释放 / DUAL 先放后占，幂等）**、
  KPI（完成/故障数、末态占用、时间跨度、故障点列表）。2D 与 3D 视图消费同一份输出→两视图口径天然一致。
- **`test_c6_replay_state.py`** — 13 个单测（占用重建/幂等/故障/crane 启发式/占用曲线/UTC-Z 时间戳/JSONL 往返），全绿。
- **`c6_sample_gen.py`** — 用**真实 orchestrator** 跑 5 任务混合调度产出同源 JSONL 并回放，端到端验证：
  入库格 1/32/54→出库释放 1→DUAL 释放 32 占 23 → 末态占用 {23,54}（手算吻合）。
- 全量回归 **139/139**（原 126 + 新 13）。

**边界**：本层只算"是什么状态"，不做任何渲染；渲染栈/版面/运镜一律未定。
