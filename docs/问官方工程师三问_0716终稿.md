老师您好!我们是参加智能仓储赛题的学生队伍,关于提交材料有三个问题想跟您确认一下,每个问题后面附了我们目前的理解,如果理解有偏差麻烦您指正:

**1. 关于初赛决赛材料一起提交的具体方式**
之前了解到初赛想拿高分可以把初赛和决赛内容一起提交。想确认:是在同一个提交包里分目录放齐"初赛三项(建模及仿真、验证报告 PPT 与 WORD、AC500 编程代码)+ 决赛两项(可视化仿真编程实现代码、演示录屏)"就可以吗?目录组织有没有指定格式?
*我们的理解:同一提交包内分文件夹放置,初赛材料完整齐全,决赛材料一并附上,无固定目录模板。*

**2. 关于决赛"可视化仿真"的技术形式**
决赛任务写的是"基于 Automation Builder 工程软件的可视化仿真"。我们已完成基于 AC500 ST 的算法编程实现,在可视化部分,我们在 AB 工程之外自主开发了一套 Web 版三维可视化仿真(HTML + JavaScript/WebGL 编写,双击浏览器即可打开,无需安装任何软件、不用联网),覆盖了赛题要求的功能:交互界面输入货物属性(序号、名称、重量、访问频次、体积)、存放位置与路径规划统计(起点、终点、路径长度、执行时间)、完成全部仓位存储总时间、仓位布局与占用情况展示;提交时会附全部源码和操作演示录屏。另外说明:开发过程中使用了 AI 辅助编程工具,仓位优化算法的设计、验证体系与工程实现由我们团队完成。想请教三点:
(a) 决赛的"可视化仿真编程实现代码"接受这种自研 Web 形式吗?还是必须基于 Automation Builder / CODESYS 可视化来实现?
(b) 如果以 AB / CODESYS 可视化为准,我们把这套 Web 版作为附加展示材料一并提交,是否允许、能否作为评分参考?
(c) 赛事对参赛作品使用 AI 辅助编程工具有没有限制或需要声明的要求?
*我们的理解:评分关注"模型可视化仿真实现"(20%)的功能完成度与效果,工具不做硬性限定;自研代码更能体现"编程实现";AI 辅助用于编码提效,核心算法与工程决策由团队完成即可,如需声明我们会在报告中如实注明。*

**3. 关于提交包体积与演示录屏规格**
因为包含演示视频,我们的提交包预计几百 MB 到 1GB。官方页要求加密压缩后经 QQ 邮箱中转站发送,想确认总体积有没有上限?另外演示录屏的时长和格式有没有要求(我们计划 mp4 格式、几分钟内讲清)?
*我们的理解:如有体积限制,我们优先压缩视频码率保证内容完整;录屏无强制规格要求。*
