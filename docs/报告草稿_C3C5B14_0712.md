# C3 · 能耗与安全叙事

## 一、年化能耗经济账

L2 混合流(H)场景下，AWRA-LS 布局较题面基线(seq)年化能耗从 546.0 kWh 降至 278.9 kWh，降幅 **48.9%**，年省电 **266.5 kWh**，折合电费省 **213 元**、CO₂ 减排 **155 kg/年**（`1_给你看\实验发现.md` F10 表 F18 行）。基于物理提升-摩擦模型（提升 mgh + 水平摩擦 μmgd，效率 η=0.85，下降不回收动能，属保守估计），沿 L2 混合流逐周期累计。

**限定说明（呼应 D-4）**：上述 kWh 是仿真侧能耗代理量（sim proxy），由堆垛机运动轨迹套物理公式折算，**并非 AC500 PLC 实测功耗**。目前无现场电表/AC500 侧功率采集可交叉验证，该能耗账仅用于策略间相对比较（AWRA-LS vs 基线），不作绝对能耗/电费的工程承诺对外宣称。

## 二、Worker-Safety 语义：重货低层 = 人工介入更安全

安全溢价账（`sim\out\oracle_gap_summary.txt:16-19`）：holistic 检测档（主打 AWRA、题面 C6 重货低层导向）相较 speed 档，重货平均层高 **0.57 vs 3.71**（低 6.6 倍）、能耗代理 **17685 vs 25891**（低 32%）。

叙事：holistic 主动牺牲约 24.7% 纯速度 gap，换重货集中低层堆放——低层意味人工/AGV 介入（取放/盘点/应急）时提升高度更低、坠落风险窗口更短，是题面 C6"重货低层"安全约束在检测器层面的直接体现；同时低层路径更短、与能耗降 32% 同向叠加，形成"安全提升与可持续提升同源共生"而非此消彼长；速度敏感场景可切换 speed 档、安全合规场景用 holistic 档、三档按场景切换（同文件 :19）。

**边界声明**：本段仅用能耗代理量与 heavy_tier 两个 sim 内部指标，**不涉及**检测器"达 oracle 98.5%"这一存在 sim lex 档/PLC holistic 档口径分歧的争议数字（详见 `docs\D1D2反验证_0712.md` / `docs\sim_plc第二层_0712.md`，本稿不引用、不定论）。

# C5 · δ（位置价值预留项）管理语义包装

**① 技术定义**：δ 是评分函数中的位置价值预留权重，默认 0.15（`docs\canonical_assumptions.md:80`），选址评分时为"未来高频存取货位"预留权重而非纯就近分配；δ=0 退化为就近贪心，δ=0.15 在高汰换率场景被验证为"重定位可行性保险"（`:113`，F21）。

**② 通俗类比**：δ 相当于给货位打分时留一手"预留金"——不把黄金位（离出入口最近、最好拿）一次性分给眼下这批货，而给未来可能高频出入的货预留。类似超市理货：不把端头架、收银台旁黄金位随手塞满当天到货，刻意留给促销期/高周转商品，避免旺季来了还现搬现挪。

**③ 可写进报告的管理语义句**：δ 权重让存储策略在"当下摆放效率"与"未来搬迁成本"间留缓冲——用小幅让渡即时最优位，换高频货未来到位时不必二次搬运的确定性，本质是仓储管理中"黄金位预留"经验法则的量化版本。

> 注：δ 的场景自适应取值（是否随密度/分布动态调整）涉主口径争议，`【待拍板·见 docs/D1D2反验证_0712.md / sim_plc第二层_0712.md】`；本包装仅描述 δ 固定默认口径（0.15）语义，不涉该争议结论。
