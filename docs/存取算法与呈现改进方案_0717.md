## 一、四问分析定稿

### Q1 存/取/存取的重要程度 → **存取联合 > 单独存 ≈ 单独取**(用户直觉正确)
赛题指标原文「存取效率(单位时间存取量)」=联合吞吐;AS/RS 行业头号杠杆=双命令周期(一趟先存后取,省约三成空驶)。项目现状:`FC_CalcDualCycleTime`(C10 口径,T23 测试)有联合行程**时间模型**,但其上**无调度算法**(配对与顺序无人决策);算法火力目前全在"单独存"。

### Q2 取货算法现状 → **无独立取货算法**(诚实盘点)
取货效率是放置质量的因变量(期望/热门取货指标即此);FB_RetrieveGood=按 ID 取的演示动作;mixed 出库序=脚本固定 paired cycles。**空白=①多件出库排序 ②入库×出库双命令配对调度(interleaving)**。机会:配对贪心骑在现成 FC_CalcDualCycleTime 上,实现量小、赛题价值最高。

### Q3 首件货送最远角 (19,0) → **非 bug,是位置保留项设计行为,但开局过激**
机理(事件0,G001 频次 1.8 冷门轻货):落第 0 层→稳定项=能耗项恒 0(均乘层高);效率项被小频次压到 0.0062;仅存保留项 δ(1-f_norm)(1-t_norm)=0.0971——频次趋 0 时退化为"行程越远罚越小"→推到最远角。设计本意"近位留热门"(C1 已证总体真赢 hot20 -6.1%),但"中远"与"最远角"的保留价值差异微弱、演示成本巨大(SEQ/NEAR 首件占 (0,0),对照刺眼)。

### Q4 看不出算法差别的原因
**浅层(呈现)**:①单页单算法靠记忆对比②差异累积中后段没人看完③角标/tieNote 是数字证据非图形分化④**货物颜色编码重量,而 SCORE 的核心贡献"热门放近列"的频次维度无视觉通道——胜利是隐形的**。
**深层(机理)**:①等时区(垂直 1/4 速,70% 格行程与列无关→多数决策进平局兜底,落位像素级接近)②**满仓守恒(三算法终态同一 267 格集合,终态占用图必然全同;可区分的只有货-格配对)**③skew 评分塌缩(效率项 6.6%,SCORE≈压层高+微弱列偏好,差异真实但秒级)。
