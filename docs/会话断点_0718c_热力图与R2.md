## 3. 本轮挖出并修掉的 3 个真 bug(都是"看着做了、其实没生效")
1. **`instanceColor` 被分配成长度 0** —— three.js 的 `setColorAt` 首次调用时按**当时的 `count`** 分配数组;代码里 `count` 初始为 0、循环结束才设 120 → 缓冲区 `Float32Array(0)`,**所有颜色写入虚空**(实测 `colorArrayLen: 0`)。这才是"色阶怎么调都不显现"的真因(不是材质偏暗)。**修法:创建 mesh 后按容量预分配 instanceColor。**
2. **热度涂在顶盖上=白涂** —— 盖 `.74×.10`,正面可见面积仅本体 `.70×.58` 的约 1/6,且被上层层板压住。
3. **2D 货位图被裁掉一半多** —— 画布 `313×313` 在 `150px` 高、`overflow:hidden` 的容器里 → 只露最上约 48%;而货物集中在**下层**(地图下半),所以左栏地图看起来"一个货物都没有"。**修法:`#slotMap` 加 aspect-ratio + max-height 自适应**(现 133×133 完整)。

## 4. 热力图最终设计(用户指正后定稿)
- **热度属于位置,不属于货物** —— 画在**货位背板**(货箱正后方 `.96` 薄板,正面看=围绕货箱的色环),**货箱本体保留等级色**(重/中/轻是稳定性/能耗评分维度,不牺牲)。
- **空位无热度**(诚实:那里没有存储活动)。用户曾提议"画在空位",经独立论证**不采纳**:只给空位上色画出的是"离 I/O 多远"的纯几何梯度,与算法好坏无关、证明不了任何事;而 02 要证的是"算法把**热门货**放到了近 I/O",证据必须是热门货**实际所在位置**。
- **冷端用中性灰**(`#aeb8c0`)非蓝 —— 货箱等级色是蓝系,冷蓝会撞色糊成一片;灰冷端让低频不抢注意力、暖/热跳出来。色阶 灰→琥珀(`#e7b800`)→热红(`#c0392b`),按 `freq_true` **秩归一化**。
- **两层分工**:总览尺度靠**热区包络**讲故事;逐格色阶是近看/放大的细节可信度层。
- **2D 货位图热力**:平面无透视遮挡,可整格铺热度底 → 补上 3D 在总览尺度必然失效的短板。
- **开关**:3D/2D 各一,DOM 挂在已有「货位热度」图例面板内 —— **刻意不动左栏栅格**(左栏每加内容都会触发一轮 @media 断点适配战,是本页最脆处)。
