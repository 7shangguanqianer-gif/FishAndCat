# DR-5 · 单人所有者 + 多 AI 代理跑超大项目的方法论（deep-research 0712）

> 问题：非技术单人所有者 + Claude/Codex 等多 AI 代理执行超大型长周期项目，业界已验证的管理方法论有哪些、本案哪些可采纳。
> 执行：deep-research，107 agents 全 sonnet/medium（0712 用户令），3-vote 对抗验证，11 条 claim 全 CONFIRMED。
> ⚠ 证据分两层采信：(a) 一手 arXiv 大样本/官方文档=「已验证」；(b) 2026 新兴个人博客/低星仓库=「灵感参考」，不得包装成已验证。另：本研究的"五维框架"是我们自造的归纳工具，非业界标准分类。

## 一、已验证层（一手实证，直接影响本案）

| # | 发现 | 证据 | 对本案的意义 |
|---|---|---|---|
| 1 | **Scope creep 是实测常见故障，非偶发**：16,118 个 agent 会话中 10.20% 自发越权，其中 66.99% 是 Scope Overreach（把窄任务当扩大编辑范围的许可） | arXiv 2605.29442 | 用户"AI 任务规划越来越混乱"的抱怨=该故障模式的直接体验；防线必须**结构化前置拦截**（任务清单固定、新活进 backlog），不能事后发现 |
| 2 | **人工纠错占已解决偏差的 91.49%，AI 自纠仅 2.99%** | 同上 | 审批门/待拍板清单不是流程负担，是系统安全的实际支撑点——不应为了速度弱化 |
| 3 | **跨会话偏差有传染性**：当前会话出偏 → 下一会话出偏概率 33.6%→51.9%（+54.46%） | 同上 | 接管提示词/复核回执/决策日志=打断错误传播链的机制，不是形式主义 |
| 4 | **CLAUDE.md 四层文件式治理是官方已验证机制**（managed→user→project→local，纯 markdown 非向量库） | 官方 docs + Dive-into-Claude-Code | 本案 CLAUDE.md/接管提示词/canonical 体系已在被验证的路径上 |
| 5 | **AI 不应自写单一真相源**：LLM 自生成上下文文件 → 任务成功率降 ~3%、成本升 20-23%；人写 → 升 ~4% | ETH Zurich, arXiv 2602.11988 | 真相源（canonical/CLAUDE.md/接管提示词）改动必须**人工拍板后**才落笔——本案红线 5 已如此，升级为明文机制 |
| 6 | **AI 终结的审查对话拒绝率显著更高**（7.1-25.8% vs 人工终结 0.9-7.8%，278K 样本/300 项目） | arXiv 2603.15911 | **"Codex 通过"≠终审、"Claude 复核通过"≠终审**——重大产出最终确认权在用户 |
| 7 | **Agent 代码技术债可测量且持续**：静态分析警告 +18%、认知复杂度 +39%，不随熟练度消退 | arXiv 2601.13597 | 轨B 写 ST 时：复杂度审查+全量测试+人工监督缺一不可；新写代码同样交 Codex 对抗审查 |
| 8 | **对抗性审计方法论**：N 个独立怀疑者试图反驳、多数存活才采信 | alexop.dev + 上条 | 本案 Claude↔Codex 双轨互审=正确路径，保持 |

## 二、灵感参考层（个人实践，样本小，可借思路不搬数字）

- mission control 面板 / waiting-for 清单 / decision queue 的具体 UI 形式（MeisnerDan/mission-control："You manage outcomes, not keystrokes"）——本案面板重构的参照原型，但**不宣称业界已验证**。
- 硬性迭代上限（单任务 8 次）/ token 预算 85% 自动暂停报告——具体数字不搬，**"设阈值+卡住即停下来问"的思路**采纳。
- 确定性编排（控制流写成代码、每步派新子代理）——本案 Workflow 已在用。
- ai-memory：git 版本化 markdown 为真相源 + 跨代理 handoff 自动注入——本案信箱协议已是手动等价物，够用。
- 反面参照：Monperrus 单人立场论文主张"AI 将取代人工审查"——与 278K 实证方向相悖，**不采纳**。

## 三、对本案的三档采纳清单

**A. 立即可用（今天生效）**
1. **用户=终审人**明文化：轨A/轨B 任何重大产出，Codex 改完→Claude 复核→**用户过目才算收口**（依据 #6）。写进面板与协作规则。
2. **真相源人工拍板制**：canonical/接管提示词/CLAUDE.md 的实质修改，AI 只出 diff 提案，用户批准才落笔（依据 #5；红线 5 升级为机制）。
3. **防 scope creep 三件套**：一次一个主线；新任务先一句话报告再动（或进 backlog）；待拍板集中一张清单（依据 #1/#2；0712 已立，现获实证背书）。
4. **决策日志带 rationale** 进面板：每个拍板记"当时为什么"（依据 #3——打断跨会话错误传染）。
5. **对抗审计保持**：N2-N5 模式不因转入执行期而撤（依据 #7/#8）；轨B 新写的 ST 代码同样过 Codex 审。

**B. 需小改造**
6. 长任务"卡住即停"：同一问题失败 2-3 次就停下来报告，不无限重试（灵感层思路+#1）。
7. ST 质量门补一项：新代码交付时附"复杂度自查"（嵌套深度/单 FB 行数），防技术债（依据 #7）。
8. handoff 文档在每个大节点强制更新（接管提示词已是，节奏固定化）。

**C. 不适用/不采纳**
9. 8 次/85% 等具体阈值数字——单一社区来源，不搬。
10. 向量库记忆方案——本案 markdown 体系已被官方路径验证，不换。
11. "AI 取代人工终审"——实证相悖，明确拒绝。

## 四、给面板设计的直接输入
- 指挥区文案加一句定位：**"你管结果，不管键盘——但你是唯一终审人"**。
- 决策日志区=一等公民（字段：日期/决定/为什么/影响）。
- 状态判定标准区列明"高风险动作审批门"（AB 互斥、docx 红线、真相源人工拍板、git 指定路径）。
- 面板底部注明本套机制的证据分层（哪些有实证、哪些是灵感）——与项目"口径诚实"红线同构。
