# D-2 量化：静态权重 vs 自适应权重对头条数字的真实影响 — 2026-07-12（overnight R3）

> 目的：用 Python sim **只切“权重”一个变量**，量化H设计口径与accel+static比较口径的差异。该比较不等于PLC数学同源或AC500真跑；PLC默认`xUseAccel=FALSE`，且ST评分分母与Python不同。
> 纯 python，不碰 AB；seed 固定（`headline.SEEDS=[2026,7,42,123,999]`），可复现。
> **一句话结论：在Python引擎内，accel+static相对H设计口径为降幅−1.6pp、取货+0.35s；8.757/60.4%/67.2%只能称Python加减速+静态sim结果，不能称PLC代理或AC500实测。**

---

## 0. 三个 Python sim 口径（均非 AC500 实测）

| 口径 | accel(加减速) | adaptive(权重) | 对应什么 |
|---|---|---|---|
| **H 报告头条口径** | True | **True 自适应** | 报告 §1.1 现行头条数字（8.40s/62%…）|
| **P Python accel+static** | True | **False 静态默认** | Python加减速+静态sim结果；不是PLC代理/真跑 |
| L 对照口径（参考） | False | False | headline.py 的"匀速+固定"文献对照列 |

> H与P只差权重，适合做sim内部消融；L=6.443是匀速+静态sim开关代理，对应PLC默认`xUseAccel=FALSE`的开关态，但仍非PLC实测。

## 1. 结果（120件/skew/sum，5-seed回归锚，±为sample SD）

| 指标 | H sim设计口径 | **P Python accel+static** | L Python匀速+静态 |
|---|---|---|---|
| awra 期望取货 | **8.402 ± 0.477 s** | **8.757 ± 0.403 s** | 6.443 ± 0.356 s |

**自检通过**：H 口径 awra 8.402s / 降 62.01% / 能耗 67.31% / 层高 0.567 与 `headline.py` 报告值 8.40/62.0/67.3/0.57 **逐位吻合** → 脚本口径正确、可信。

## 2. ★ D-2 影响（H → P，只切权重一个变量）

| 头条指标 | sim H | Python accel+static P | 差 |
|---|---|---|---|
| awra 取货时间 | 8.402 s | 8.757 s | **+0.354 s（+4.22%，变慢）** |
| 较 seq 降幅 | 62.01 % | 60.41 % | **−1.60 pp** |
| 能耗降幅 | 67.31 % | 67.19 % | **−0.12 pp（≈不变）** |
| 重货平均层高 | 0.567 | 0.550 | **−0.017（更低=更安全）** |
| 违规Σ | 0 | 0 | **0（无安全回退）** |

**解读**：
- 自适应权重（D-2 的机制）对取货头条的贡献 **只有约 1.6pp / 0.35s**——远非"数量级"差异。
- 能耗降幅几乎不受影响（−0.12pp）。
- 重货层高在静态权重下**反而略低（0.550<0.567）= 略更安全**——因自适应权重更激进地压时间、略微牺牲重货层高；静态权重更保守。这是诚实、可讲的权衡，不是纯损失。
- **约束违规两口径都为 0**：静态权重不引入漏件/超限风险。
