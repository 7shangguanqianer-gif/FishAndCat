## 六、S1 原始发现(算法/仿真/PLC,带 file:line)

- **算法**:AWRA-LS=Rank(λ=0.50/0.35/0.15)→Assign→LocalSearch(重定位+成对交换)三段(warehouse_sim.py:316-388),标准构造式启发式+2-opt,正统不新颖,理论上界弱(仅局部最优)。真对标 Hausman/Schwarz/Graves 1976 三分类+CB(70/90分界)+TOB(COI 变体,strategy_lib.py:96-141)七策略矩阵。SCORE 是 class-based 连续化推广,未用乘积形 COI(TOB 内部有但检测器主线未当主力)。
- **理论**:analytic_bounds.py 三级(离散精确期望 vs 仿真;Bozer-White 1984 连续近似 vs 离散;频次感知松弛下界=重排不等式);lap_optimal.py 用 linear_sum_assignment 精确最优(货物间无交互=LAP 非 QAP 多项式可解,判断正确)。oracle_gap.py:68-78 gap 只在 excess_fail=0 可比域算(避免漏件虚假优势),Rice/ASlib 框架,VBS vs SBS 区分,C13 双口径均非理论上限(口径克制)。
- **检测器**:查表式规则(4 阈值,strategy_lib.py:173-191),30 seed 标定;adaptive_weights.py 自曝「200/240件×三分布 6 个 online 单元全 INFEASIBLE」(canonical:96)=高密度在线无可行解,真实短板但如实披露。
- **PLC**:算法完整移植非简化;04_FB_Warehouse.st 分片(nPairsPerCycle=200/nBatchPerCycle=10 防看门狗);tie-break/FC_CheckFeasible/FC_CalcScore(03_Functions.st:204-231)与 Python 逐项对齐,导出向量容差 1e-3 比对;06_PRG_Test.st N_CASES=77 全真断言无占位。**取货侧简化**:FB_RetrieveGood(04_FB_Warehouse.st:1163-1173)只按 ID 取单件演示,FC_CalcDualCycleTime(03_Functions.st:146-161)只有时间模型无调度算法(项目自审 存取算法与呈现改进方案_0717.md:9-13 已承认)。
