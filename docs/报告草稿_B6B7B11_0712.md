# B6 · §4.5 情景参数稳健性 + 免费改进池条目

## §4.5（一）情景参数稳健性

**方法**：针对 canonical 标注"情景假设-显式声明"的三轴——G2 到达强度（峰谷利用率 ρ）、G3 频次估计噪声（σ）、G4 故障画像（MTBF）——补做单参数扫描（OFAT），检验"AWRA-LS 布局优于顺序基线"是否对情景取值敏感。复现脚本 `sim\scan_g2g4.py`，汇总数据 `sim\out\scan_g2g4_agg_data.csv`、参数与元数据 `sim\out\scan_g2g4_agg_params.csv`，逐 seed 明细 `scan_g2g4.csv`（840 行）。扫描点（`docs\G2G4敏感性扫描_0710.md:11-15`）：G2 ρ∈{0.60,0.75*,0.90}、G3 σ∈{0.15,0.30*,0.45}、G4 MTBF∈{1500,3000*,6000}s（*=canonical 基准）。引擎零改动复用 `realism.py::run_tier3`，30 seed(2026–2055)×7 点×4 策略，共同到达流（CRN）+ 配对 t 检验。

**结论**：全 7 配置点上 AWRA-LS 相对顺序基线的出库均时优势 100% 稳健：
- base：awra 10.931s vs seq 20.583s，降幅 46.89%，t=−44.04***，30/30 配对全胜（`scan_g2g4_agg_data.csv:2`）
- G2 ρ=0.60/0.90：降幅46.89%，共同到达流下AWRA P95约376→527s（seq 675→1116s）
- G3 σ=0.15/0.45：降幅 47.06%/46.89%，t=−44.24/−43.66***（`:5-6`）
- G4 MTBF=1500/6000：降幅46.89%，共同到达流下AWRA P95约770/348s（seq 1365/668s）

机理：每个配置点/seed以seq reference生成一次外生arrivals，四策略共享；出库均时是纯行程口径，G2/G4不改行程几何，G3噪声经布局间接影响。旧版按每策略自身服务均值反推λ的P50/P95已废止。

**诚实边界**：新扫描中AWRA<seq的P50配对优势率七点均100%，P95为93.3%–100%；这只支持AWRA相对seq，不支持AWRA相对near/score全面最优（canonical单seedP95是near 651s < AWRA 667s）。

**图表指向**：G2/G3/G4 三组各绘"降幅%（左轴近水平线）+ P95（右轴单调折线）"双轴图，呈现"均值免疫、分位单调"；逐 seed 明细可作配对优势率箱线图。

> 口径：`canonical_assumptions.md`继续把G2/G3/G4标为“情景假设-显式声明”；0712共同到达流扫描只增加敏感性证据，不把假设升级成现场标定值。

## 一、吞吐口径边界声明（230单操作/h≈115双周期/h）

**是什么**：`230 次/h` 出自 `1_给你看\实验发现.md:17`（F18表F8行，sum口径，替代切换前268次/h）。计算见 `sim\mixed_ops.py`：`n_ops=2*len(new_goods); thr_dual = n_ops/(tot_dual/3600)`。`tot_dual`是默认100个“存1+取1”双周期的**运动学sim行程时间**累加。故230次/h表示约230个**单操作/h**，等价约115个双周期/h；不是230个双周期/h。固定SEED=2026，单次运行、单负载序列，无CI。

**不是什么**：
- **不是 AC500/AB 实测吞吐**——不经 PLC 扫描周期(10ms)、OPC 写值、I/O 轮询、看门狗、分片开销（这些只在 L4 口径出现）。
- **不是多周期统计量**（无 CI、无多 seed）。
- **不是档3（泊松到达）下成立的量**——`现状与任务.md:281`（深思#2）指出泊松下"吞吐=到达率"、不再反映布局优劣；230 次/h 只在档2（连续满载假设）有意义。
- **不是"AC500 实测吞吐 230 次/h"**——属 D-4 已标错配点，不入报告定论区。

**报告安全表述**（建议在 §3 头条区 + §4.5 拟真阶梯区加边界句）：
> "吞吐230次/h是H运动学sim在连续满载假设下的单操作容量（约115双周期/h），忽略控制器扫描、通信、装卸、排队、故障与机械限位，较基线+75.2%；并非AC500在线完成率。控制器侧只有AB PC仿真的任务周期代理，另列且不互证。"

若需静态消融对照，可用`docs/D2量化_0712.md`的**Python加减速+静态sim结果**8.76s/60.4%/67.2%；它不是PLC代理或实测，更不能与230次/h拼成“AC500实测”。

# B11 · 数字孪生成熟度阶梯（决赛展望）

本项目技术路线天然分层：Python 建模仿真产出算法/数据、AC500(CODESYS 系)ST 承载控制逻辑、AB 画面+Factory I/O 承载可视化与物理验证——恰对应数字孪生成熟度递进。

## 三层阶梯（文字版结构图，供配图）

```
L0  设计时仿真（Design-Time Simulation）
     Python 建模 + 统计实验，产出策略、参数、上界
     └─ 路径：ST 移植（算法/评分函数 → AC500 ST）
                ▼
L1  运行时画面级孪生（Runtime Visual Twin）
     控制逻辑运行在 AB/CODESYS，画面实时渲染 PLC 内部状态
     └─ 路径：控制信号改走 OPC UA 连独立物理引擎
                ▼
L2  运行时控制在环（Runtime Control-in-the-Loop）
     控制器 ↔ Factory I/O 物理引擎经 OPC UA 闭环，
     驱动"会掉/会堵/会被挡"的虚拟工厂而非渲染函数
```
