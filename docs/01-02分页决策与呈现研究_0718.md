# 01/02 分页决策与呈现研究 · 2026-07-18

> 触发:用户问「为什么分 01/02 两页?两页版面一致评委怎么看出差异?要不要全部重新设计?」
> 方法:3 个 sonnet agent 并行——① 仓库考据分页原始理由;② 上网 deep-research AS/RS 存取作业呈现惯例;③ 上网 deep-research 多能力演示可辨识呈现。
> 一句话:**分页是对的(赛题硬要求),但「把 02 改得和 01 不一样」方向错了——研究指向相反解法:补齐同壳 + 内容层显式标记差异。**

## 1. 为什么当初分两页(考据结论,带出处)

- **原始动因=赛题硬指标**:0716 用户验收质问「算法只有存放吗?取出是赛事要求吗?」→ 盘点赛题原文行 22「存取效率(如单位时间存取量)」实锤「存」与「取」并列要求。当时唯一活跃的 fill 页只演示「存」半边,遂拍板双页并存分工(`docs/存取效率设计_素材包_0716.md:1-13`、`docs/续窗交接_0716B.md:24-25`)。
- **01 证明**:空仓 0→267 逐件放置决策与容量推进——算法分化「30 秒肉眼可见」的主秀。技术=`sim/fill_only.py`(单向入库、不产出库事件、SHA 级数据隔离)。
- **02 证明**:存取交替稳态作业——入出闭环、取货侧红利的唯一 Web 实测兑现场。技术=`sim/mixed_ops.py`(120 件初始入库 + 100 周期每周期 1 存 1 取、单命令 T=2·t(存)+2·t(取) vs 双命令 T=t(存)+t(存→取)+t(取) 两口径)。
- **∴ 02 不是「01 换皮」**:fill_only 与 mixed_ops 是两种真实不同作业模式,双命令节省率 28-31%(几何红利·杠杆一)与出库均时布局差异(52%·杠杆二)是 01 完全不涉及的独立技术维度(`docs/双命令调度设计稿_0718.md:8-38`)。
- **决赛四件** = 00_导览 / 01_连续填仓 / 02_入出闭环 / 03_FIO物理证据(`docs/续窗交接_0717.md:37`)。

## 2. 关键前提纠正(用户与我先前的判断都需修正)

**用户担忧的「01/02 版面一致」并非当前现状,而是既定终态目标——且这个终态是对的。**

- **现状不是「雷同」,是「代差」**:02_入出闭环.html(1108 行)与 0714 冻结的「三方向候选」样张**只差 2 行**(title + AUTO 文案),尚未套 01 的 A 壳/治理叙事;01 已被大改(466 行治理叙事 diff)。即 **01 武装到牙齿、02 还穿旧衣服**,两页目前是不同代,不是长得一样。M4b 冻结规格(`docs/mixed升A壳_冻结规格_0717.md:11-19`)本就要求 02 **复用 01 的 A 壳栅格变量**(`--rail-w`/`#topbarA`/`#workHud`),只换货流内容——这条尚未执行。
- **「同壳」是特性不是缺陷**(小倍数原理):Tufte 小倍数核心=一致的设计让观众注意力落在**数据变化**而非图形设计变化上。AS/RS 学术界画单命令 vs 双命令也是**同坐标系同比例的并排子图 (a)(b)**,差异只体现在轨迹本身,不换布局/视角/配色。→ 把 02 改成和 01 不一样,反而**违背**呈现惯例,还工作量大风险高。

## 3. 研究三方汇聚的结论

**A. 分页 vs 合并 → 保持分页,别合并**
- AS/RS 数据/KPI 层惯例本就按作业阶段分模块(WMS putaway/picking 分屏);评委需要「可比性」,分页提供并排对比空间。
- **明确反对「01/02 合成一页 + 模式切换器」**:toggle/segmented control 的用户语义是「同一内容换个展示」,会**坐实评委「这俩是一回事」的误判**,与凸显技术增量的目标相反(NN/g、SAP Fiori、tabs-vs-segmented 语义分析)。
- 本项目 `docs/双命令调度设计稿_0718.md:40-45` 早已从另一角度建议不做页面级单/双命令切换(口径解释负担 + seq 基线节省率反直觉易被误读)。

**B. 3D 物理层几乎不分页,但本项目分页有其道理**
- 数字孪生/仿真(AnyLogic/FlexSim/Demo3D)通常单一连续 3D 场景同时跑存/取两流,靠颜色/形状区分——因为物理上一条巷道一台 S/R 机,存取本在同一轨迹同一时间轴。
- 但本项目分页是**为叙事分工**(01 讲放置智慧、02 讲取出兑现)+ **赛题存取并列要求**,不是物理仿真复刻,故分页成立;代价是必须靠 C 部分手段消除「重复感」。

**C. 让差异「一眼可见」的成熟手法(小倍数框架内加显式标记)**
1. **同壳一致框架**:两页复用同一镜头脚本 + HUD 布局,唯一变量是轨迹动画本身(AS/RS 并置子图惯例)。
2. **02 内容层标记独有环节**:取货 lane、载货腿实线/空驶腿虚线、**高亮双命令独有的「存货位→取货位」交织腿**;给「01 没有的环节」(取货/路径/二次定位/库存回写)加专属高亮色/标注框(小倍数「显式画出变了什么」)。
3. **量化对比角标**:双命令一次往返完成一存一取 vs 单命令两次往返,空跑占比从约 1/2 降到约 1/3、**省约 33%**(Bozer-White 1984 SC≈1.27/DC≈1.71/2×SC=2.54;Graves-Hausman-Schwarz 1977 交织≈30%;Eynan-Rosenblatt 1993)。用业界公认数字逻辑,比自造话术站得住。
4. **能力对比总览(小倍数并置)**:两页之上/之间加一环节,把 01↔02 关键指标并排(状态节点/新增控制回路/取货侧模块/算法维度),颜色标记 02 多出的部分——评委不用脑内做差值。
5. **口头讲解用 before-after 锚**:「在刚才入库演示基础上,新增了 XX 取货环节」——清晰语言锚胜过界面区分度。

**D. 「demo 疲劳」是真风险(用户担忧不是过虑)**
- 评委连续看多项目会审美疲劳;两页版式雷同若**不加显式差异标记**极易被判「重复」而非「递进」。→ 但解法是「显式标记」不是「改得不一样」。
- 提醒:ABB 杯技术评价实质抓手是控制算法能力(位置闭环/振动抑制等),非界面观感;界面服务于让算法增量可见(ABB 官网 2024 榜单)。西门子杯评分细则未公开,留待后续 Q4 深挖。

## 4. 给决策的净结论

**别「全部重新设计 02 成与 01 不同」——那违背小倍数/并置惯例、工作量大风险高。正确路径:**
1. **先把 02 补齐到与 01 同 A 壳**(执行 M4b 冻结规格,消「代差」)——这是收口,不是镀金。
2. **在同壳框架内,用内容层显式标记 02 的独有技术增量**(取货 lane / 交织腿高亮 / 省一趟角标 / 01↔02 能力对比总览)。
3. **口头/字幕用 before-after 叙事**锚住「02 比 01 多做了取出闭环」。

这样既解「评委看不出差异」的真风险,又比推翻重写省得多、稳得多,还踩在业界惯例上。
