## 两个 tie-break/阈值 gap 详析

**G1 — assign 次级 tie-break**:`empty_feasible_slots` 为 col 外 tier 内扫描序。
Python assign 用四元组(平局按 tier→col),ST 用 (score,travel)+扫描序(平局按 col→tier)。
仅当两个可行位 **score 与 travel 同时相等**时选位不同——离散格点下 travel 相等常见
(如 (4,0)/(0,1)/(2,1)/(4,1) 均 travel=2),但叠加 score 相等为构造性巧合。
- **实测(consistency_probe.py,tie_break 参数隔离)**:演示批 20 件 + 12 场景×30 seeds
  = 390 次运行,**布局分歧 0 次**。gap 理论存在、真实数据零触发。

**G2 — LS 改进接受阈值**:Python 双精度用 `1e-12`,ST 单精度用 `1e-6`(REAL 精度
~1e-7,单精度下 1e-12 是噪声)。改进降低量 δ∈(1e-12,1e-6) 时 Python 接受、ST 拒绝 →
可能收敛到不同局部最优。
- **实测(consistency_probe.py,ls_eps 参数隔离)**:G2 单独触发 **35/390**(高密度为主,
  dense_240 seed2032 达 26 件);G1 tie-break 0/390。**但分歧布局的聚合指标 exp_t/energy
  差异 = 0.00(完全相等)**——sim 与 ST 收敛到 score 完全等价的不同局部最优(高密度下
  等价最优解有多个分支),非缺陷。**演示批 20 件(T25 用同批)零触发**,逐位护栏有效。
- **定位**:sim↔ST **数值等价**(聚合指标逐 seed 完全相同);演示批逐位一致。G2 是单双
  精度结构性差异,不可靠改 ST 阈值消除(REAL 精度 ~1e-7 下更小阈值无意义);报告表述用
  "数值等价 + 演示批逐位一致",不宣称"高密度全场景逐位相同"(那是假的,且没必要)。
