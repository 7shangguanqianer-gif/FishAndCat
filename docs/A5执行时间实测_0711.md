## 判读(报告 §5 与 A1 硬实时论证的实测地基)

1. **单件推荐 ≤1 周期(10ms)**:这是 A1 oracle 实验"硬实时约束"论证的地基——若在线并行
   试跑 7 策略再择优,需 7 倍计算+统计窗口,违反入库节拍;查表式检测器(近零延迟)是该
   约束下的工程最优形态(检测器 lex 档达 oracle 98.5%,见 fig13)。
2. **400 库位规模批量优化 20.6s(240 件,90% 填充)全程非阻塞**:分片状态机每周期只做
   nBatchPerCycle 件/nPairsPerCycle 对,不触发看门狗——"优化算法跑在实时控制器上"的
   关键工程证据;分片参数可调(S240b:加大分片 → 18.8s,余量可继续放大)。
3. LS 局部搜索是周期数主项(S240 vs S120 超线性=O(n²) 交换对数),与 README_PLC §4
   的解析估算一致——实测与理论互相背书。
4. 边界:AB 仿真跑在 PC 上,真机行值待借到实体 AC500 补测(报告口径写
   "仿真实测+真机标称预估",与既有 L4 口径一致)。
