## 1. 五题结论(证据分级)

### ① 3D 逐格热力呈现 → 半透明前罩方案成立(已实施 `2bc0d8e`)
- 【一手·three.js issues/forum】透明+InstancedMesh 无内建实例排序(issue #27170 仍 open);
  `depthWrite:true` 透明区渲成黑块;`depthWrite:false` 后 draw order 可压过真实深度序;
  透明物在 opaque pass 后渲染(同深度必盖不透明物);`alphaHash` 可免排序但有噪点。
- 【本地推理,一手约束下成立】我们的前罩全部共面同深、实例互不重叠 → 排序限制不适用;
  `depthWrite:false` 防黑块;透明后渲=正确叠在货箱前。实测:opacity .45 黄罩×蓝箱混绿失真 →
  **.78 热度主导**;前移至 `loadY-.40` 盖过 lid 前脸露边。
- 【类比·产品】Tecsys「3D heat map digital twin」、Slot3D「CAD 色阶按拣选速度给库位着色」——
  行业方向=「按库位着色热力」(与我们同向),但全为营销级描述、无渲染技术细节(claims 自证)。

### ② 深色遥测块 → 白化统一(已实施 `ac6ca7c`,用户先行拍板)
- 【一手·Rockwell ISA-101 对齐白皮书(PDF)】全屏统一非饱和浅灰底(#E0E0E0 级);
  动态遥测数据规定「底色与显示背景一致 + muted blue(#475CA7 级)数据色」;
  **黑/深底唯一被允许的场合=编码故障/坏配置语义**,不作风格点缀。
- ⚠ 诚实标注:此组 claims 第一轮 verify 3-0 全否、第二轮转 confirmed——**两轮裁决打架,置信降档**。
  但方向与用户拍板(「统一成白色」)一致,且我们的条色 #17365d/#4a6fa5 本就是 muted blue 系,采信实施。
- 我们未用黑底表故障(红系 banner/红纹),与「深底=故障语义」条款无冲突。

### ③ 双视图切换足迹一致 → 已实施(`421aa96`)有实证支撑
- 【一手·IEEE InfoVis 2001(PNNL)】change blindness 在可视化状态对比中实证存在,大变化也会漏检;
- 【一手·受控实验(n=24)】切换时**保持恒定参照(轴/尺度不变)**显著降低估读误差与「说不清」应答——
  等级块放大到近整格(足迹恒定)正是该结论的落地;动画过渡(Gemini/IDL)有据但对低频 toggle 属过度工程,不做。

### ④ 停机时长标注 → 拆分显示正中工业口径(已实施+本批补比例条红纹)
- 【多源趋同·OEE/ISO 22400-2】downtime=独立 Availability loss 类目(设备故障/换型两类),
  **不得折叠进作业/周期时长**;cycle time 应纯为增值过程时长,停机单列有原因码;
  西门子 MindSphere(一手产品文档)机器状态 timeline=按状态分色。
- 我们的「9.7+600s」红标拆分 + 比例条故障段红纹 = 该口径的直接落地;
  原「609.7s 无标注」正是被该口径否定的呈现。

### ⑤ 暂停帧遥测显示 → 无行业一手,按物理口径自定并披露
- AVEVA trend/replay 文档两条源均未抓到有效 claims——**该题一手证据缺失(负结果如实记)**。
- 采用的工程口径:seek 冻结帧显示该帧瞬时值 + **故障保持位速度清零**(物理事实,`421aa96`)+
  显眼「⏸ 已暂停」贴条声明帧性质。无标准反对证据,口径已披露。

## 3. §7 deep-research 占位符三连事故(机制记录,防复发)

- 现象:同一 workflow 的 synthesize 单枪三次返回占位符(`summary:"test"` → 加固 prompt 重跑 →
  `"test debug call to verify findings array parameter parsing works with schema"`)。
- **机制根因假设(第三次占位文本自证)**:synthesize agent 先发「试探 schema」的 StructuredOutput
  调用,而 workflow 把**首次调用当终值**收走——prompt 加固无法根治(不是模型偷懒是调用时序)。
- 连带教训:**resumeFromRunId 时 args 必须与原次逐字节一致**,否则 scope 枪缓存失效 → 下游 prompt
  全变 → 全量重跑(本次实测:第二轮未走缓存,4.4M tokens 重烧)。
- 已做:主脚本 `~/.claude/workflows/deep-research.js` synthesize prompt 加 HARD RULE 禁占位符
  (对时序问题无效但无害);**处置惯例改为:synthesize 占位时不再重跑,直接由主会话从
  journal.jsonl 的 claims 综合**(扇出产出通常完好,综合本就是主循环角色)。
