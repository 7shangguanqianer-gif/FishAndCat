## 2. 落地方案:按 ISA-101 分层拆 `insightStack`

实测左栏(`scratchpad/probe.mjs`,1600×900):内容 **914px** > 可用 **850px**,其中

| 块 | 高度 |
|---|---|
| `insightStack`(队列 + 6 格本批统计 + S1 对照 + 证据链) | **537** |
| `slotMapWrap`(2D 货位图) | 150 |
| `decisionWrap` | 110 |
| `phaseRail` | 80 |

**拆法(对应 ISA-101 分层)**:

| 内容 | 层级 | 处置 |
|---|---|---|
| 运行队列、本批统计 6 格 | **L1 常驻** | 留左栏,压缩高度 |
| 当前作业/相位 | **L1 常驻** | 留(`phaseRail`) |
| **S1 E02 基线对照**(4 条横条) | L2/L3 | **降级**:折叠(accordion)或移到顶栏 chip → overlay |
| **本屏证据链**(3 条) | L3/L4 | **降级**:同上 |

预期:537 → 约 240px,左栏总高 914 → **约 610px < 850px**,**不需要滚动条**。

## 3. 与热力图方案的合流(两条研究线在此汇合)

`docs/热力图总览层研究与自我更正_0718.md` 的关键重新框定:
> 2D 货位图「每格 6px 不可读」的根因**不是可视化方法**,是画布只有 133×133 —— 而 133px 是左栏空间不足逼出来的。

**拆掉 S1 对照 + 证据链腾出的约 300px,正好给 2D 货位图。**

| | 现状 | 合流后 |
|---|---|---|
| 2D 画布 | 133×133 | **约 280×280** |
| 每格边长 | 6.6px | **约 14px** |
| 逐格色阶可读性 | 不可读 | **可读**(且有 warehouse-science 逐格上色的一手先例支持) |
| 是否需要发明聚合算法 | 本以为需要 | **不需要**(决策梯①:这功能需要存在吗) |

**3D 侧仍需分层**(透视遮挡 + 25px/格是真实约束):细节层=货位背板色阶(现状),总览层=分区聚合(ArcGIS aggregation 有依据),**弃用**证据已垮的「Golden Zone 热区包络」说辞。
