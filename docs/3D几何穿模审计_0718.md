### 根因(代码级)

`02_入出闭环.html:1075-1089`
```js
function addTransferBridge(from,to,material){          // ← 无 width 参数,硬编码 .62
  const dx=to.x-from.x,dy=to.y-from.y,len=Math.hypot(dx,dy);
  const bridge=new THREE.Mesh(new THREE.BoxGeometry(len,.62,.13),material);
  bridge.position.set((from.x+to.x)/2,(from.y+to.y)/2,.095);   // z=.095
  bridge.rotation.z=Math.atan2(dy,dx);                          // ← 斜置
}
addTransferBridge({x:INFEED.loadX,y:INFEED.y},TRANSFER_ANCHOR,M.infeedRail);
addTransferBridge(TRANSFER_ANCHOR,{x:OUTFEED.startX,y:OUTFEED.y},M.outfeedRail);
```
02 的两条输送带都在 `y=STATION_Y=IO.y-1.75`,而 `TRANSFER_ANCHOR` 在 `y=IO.y`。
→ 桥必须**斜跨 1.75m**,`rotation.z≈±63°`,AABB 膨胀到 `1.45×2.03`,横扫整个 I/O 区。
→ 且桥板 z 范围 `.03~.16`,而滚筒 z `.105~.195`、侧轨 z `-.01~.17` → **必然互穿**。

## 3. 修复方案(待执行,按性价比排序)

**A. 转运桥改轴对齐折线(治 02 独有病,收益最大)**
把「斜跨一根板」换成「沿 X 一段 + 沿 Y 一段」的 L 形,两段都轴对齐;并把 z 压到低于带面(`z≈0.02`,读作地面转运台而非悬空板),宽度从 `.62` 收到 `≈.34`。
→ 一次性消掉 6 类约 10 处相交,且视觉上从"莫名斜板"变成"合理转运台"。

**B. 复用 01 的 `width` 参数化签名**(决策梯②复用勿重写):把 01 的 `addTransferBridge(from,to,material,width=.62)` 搬到 02,两页函数统一。

**C. 轮子外露**:堆垛机行走轮 z 下移或底座抬高,使轮子露出约一个半径。

**D. 顶部滑轮组去重**:提升机顶三件重叠,保留一件或错开 z。

**E. 柱帽/桥板**:桥板改 A 后柱帽相交自动消失(桥不再覆盖柱位),复核即可。

> **执行纪律**:每改一项必须重跑 `geomaudit.mjs` 对比 pairs 数下降,并多角度截图 + 裁剪放大复核;01/02 同步改;改完跑 boot-QA(02 须 8/8,01 须 46 绿)。
