## 4. 为什么**不**建议「全部重新设计排版」

1. **同壳是已被研究支持的特性,不是 bug**。推翻同壳 = 推翻 D4/D5 与小倍数论证,并坐实评委「两页是一回事」的误判风险(该风险正是当初否掉「合并成一页 + 模式切换器」的理由)。
2. **真正的问题是漂移,不是设计**。C1-C3 是三处具体的、代码级可定位的不同步,补齐即可,成本远低于重做。
3. **重做会丢掉本轮刚建立的证据链**:02 左栏是按 ISA-101.01-2015 四级分层做的(deep-research 3 源 3-0 通过),推倒重来等于把有依据的东西换成没依据的。
4. **时间账**:距 8/15 自定目标不足 4 周,R3 录屏 / R4 报告 / R5 导览三项收口都**依赖 01/02 定稿**。壳体重做会让这三项全部推倒重排。

**建议:做「一致性补齐」(C1/C2/C3/C5),不做「全部重构」。** 差异严格限制在内容层。
