## 1. 开工自检(缺一项即停,报告用户,不硬上)

**能力三项**(对自己的 GUI 工具逐项确认):
1. 能截图看到 AB 窗口内容(非黑块);
2. 能鼠标:单击/双击/右键/**按下-移动-抬起分步拖拽**;
3. 能键盘输入含 `$ [ ] . , #` 的字符串(先在无害处试打 `$FIRSTDIM$` 验证,特殊字符
   常走剪贴板通道,需相应授权)。

**环境五项**:
1. **只用现有 AB 实例**(任务栏已有)。**严禁**用"打开应用程序"方式唤起 AB——会开出第二个
   实例(Start Page 空窗口),多实例曾造成混乱。若发现多实例:只留标题含 `ABB_WH.project` 的。
2. **AB 保持窗口化,不最大化**(最大化后工具箱移到屏幕右上,会被一个置顶浮窗挡住)。
3. **离线状态**:标题栏/状态栏无 SIMULATION 在线标志。在线时画面编辑器只读——先 Online → Logout。
4. 画布上可能遗留 1 个无属性裸矩形(名 GenElemInst_2,前次实验残留):**选中→Delete 删掉**,从零开始。
5. 工具箱(ToolBox)面板点开 **Basic** 分类要能看到 10 个元素缩略图(Rectangle 等)。
   看不到=面板太矮(11 个分类标签挤满了):菜单 **Window → Reset Window Layout**,或把工具箱拖高。

## 2. 坑库(前人 computer use 实测踩过;现象→真相→对策)

| # | 现象 | 真相 | 对策 |
|---|---|---|---|
| 1 | 截图里出现规整的纯黑矩形 | 自动化工具的隐私遮罩把非授权窗口涂黑,**不是 AB 故障** | 无视它或请用户关掉那个置顶浮窗;别对着黑块诊断半天 |
| 2 | 工具箱元素拖到画布没反应 | CODESYS 不认快速拖放 | **单击缩略图(变蓝选中)→ 到画布上"拖画"一个区域**:按下左键→分多步慢移→抬起 |
| 3 | 属性面板找不到 Color variables | 高级属性默认隐藏 | 勾属性面板右上角 **Advanced** |
| 4 | 双击属性行没进编辑框/点错行 | 展开属性组后行会**下移**,旧坐标失效 | 每次展开/折叠后**重新截图定位**目标行再双击 |
| 5 | 打字报"自身窗口占用焦点"类错误 | 自动化端自己的窗口抢了键盘焦点 | **双击目标编辑框+打字放在同一个不可中断的批次里**连续执行 |
| 6 | 打字显示成功但单元格仍是空 | 特殊字符走剪贴板粘贴,无剪贴板写权限时**静默失败** | 先补剪贴板授权;打完必须**放大截图核实**文字真的进去了 |
| 7 | 右键菜单没有 Multiply visu element | 选中的不是矩形本体 | 先单击矩形确认选中框出现;再不行去顶部 Visualization 菜单找;都没有→截图停下汇报,**禁止手工复制 400 个** |
| 8 | 属性改完不生效 | 编辑框没提交 | 输入后按 Enter;再截图核实 Value 列已显示新值 |

**通用纪律**:每做一步→截图核实一步;同一步骤失败重试**至多 2 次**,仍失败→截图+现象记录,
跳到"止损汇报"。禁止在失败处换野路子乱点。

## 3. 批次与检查点

> 每批次末尾:**Ctrl+S 保存工程** → 全窗截图 → 向用户汇报「批次X完成/卡在第几步」。
> 具体元素坐标/属性值/变量名,一律照卡执行,此处只列骨架与检查点。

### 批次 A:400 格网格(实验判据;【原卡】§3)
1. 双击设备树 Application 下的 **VisuMain** 打开画面编辑器(已存在,不新建);
2. 按【原卡】§3.1 放模板矩形:Position X=40 Y=40 W=28 H=20;
3. 按【原卡】§3.2 绑颜色变量:属性 **Color variables → Normal state → Fill color**,填
   `GVL_Visu.VisuSlotColor[$FIRSTDIM$, $SECONDDIM$]`(勾 Advanced 才看得到该属性;逐字,含美元号);
4. 按【原卡】§3.3 右键矩形 → **Multiply visu element**:第一维 20 个/偏移 28,第二维 20 个/偏移 20;
5. 按【原卡】§3.4/§3.5 加行列标注与图例(6 静态色块;**另加第 7 块:青 16#26C6DA=查询高亮**,
   这是块3 新增状态 6,原卡未列)。
- **检查点 A**:画布出现 20×20 无缝网格,右缘 x≈600 下缘 y≈440;保存后标题栏 `*` 消失。
- **A 卡死即止损**:典型死点=第 3 步占位符打不进(焦点/剪贴板双坑)或第 4 步 Multiply 对话框
  字段对不上。截图汇报,建议用户转"手搭+导航"路线,**不要继续 B/C/D**。

### 批次 B:堆垛机 + 右侧面板(【原卡】§4/§5 + 【增补卡】§1.1)

## 4. 红线(违反任何一条=事故)

- **只动画面对象**(VisuMain/VisuStats/Visualization Manager/Alarm Configuration/WebVisu)。
  **禁止**改设备树里任何 ST 对象(GVL/POU/PRG)、任务配置、设备参数。
- 变量名/占位符/颜色值**逐字照卡**,打完放大截图核对;绑错变量比不绑更糟(在线才暴露)。
- Multiply 对话框字段与卡对不上→截图停下汇报,不猜(原卡§9 第 1 条)。
- 不删不改工程里既有对象(除 §1 环境五项第 4 条指名的遗留矩形)。
- 在线(SIMULATION)状态不动画面编辑器;编辑前确认已 Logout。
- 卡死重试上限 2 次;止损汇报优于带伤推进。额度是有限资源,A 批次若耗时超过约 1 小时
  仍未过检查点,同样触发止损。
