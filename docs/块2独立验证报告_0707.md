## 3. 发现（按严重度；每条区分实测/推测）

### 🔴 F1【假绿·中】gap 表"聚合零差异 / 完全相等"被实测证伪 —— 任务书批判点1命中

- **【实测·文档声称】** `docs/T2.1_一致性审计_gap表_0707.md:37-38`：
  > "分歧布局的聚合指标 exp_t/energy 差异 **= 0.00(完全相等)**"
  > "sim↔ST **数值等价**(聚合指标**逐 seed 完全相同**)"
- **【实测·探针输出】**（§2.1 第11行）：`exp_t 差异上界 1.09e-02，energy 相对 7.18e-04` —— **非零**。

- **【实测】** 35/390 场景布局分歧真实（261 件货位不同，最大 dense_240/seed2032 差 26 件——这些数字 gap 表都对）。
- **【推测】** 大结论"sim↔ST 实质等价"**不倒**：exp_t 差 0.011s（相对约 0.1–0.5%），最大差异在 `dense_240`（90% 填充极限压力场，**不进报告头条**）；演示批/T25 口径实测 **0 差异**。故这是**文档 over-claiming，非算法 bug**。头条口径 `L120_skew` 未被探针单列，但其密度远低于 dense_240，分歧幅度【推测】更小。
- **建议修正**：gap 表"= 0.00(完全相等)/逐 seed 完全相同" → "分歧场景 exp_t 差 ≤1.1e-2 s、energy 相对 ≤7.2e-4（工程可忽略）；演示批/T25 逐位零差异"。

### 🟠 F2【隐患·中低】gap 表"reloc tie-break 已一致"不准，且探针有覆盖盲区

- **【实测·代码不一致】**
  - ST 重定位 `plc/04_FB_Warehouse.st:298`：`IF rS < rBest - 0.000001`（扫描序**逐步下降**，保留第一个达标位）。
  - Python 重定位 `sim/warehouse_sim.py:355`：`best = min(slots, key=score_fn)`（取**全局最小**位）。
  - 多个空位 score 彼此差 <1e-6 时，两者选位可能不同。而 gap 表 `T2.1...gap表:15` 称 reloc "tie-break 一致"。
- **【实测·探针盲区】** `consistency_probe.py` 的 `"st"` 模式只切换 assign 的 tie-break 与 LS 的 `ls_eps`（`warehouse_sim.py:328-331` 的 `assign_key`），**reloc 内部始终是 Python `min(key=score)`（:355，不受 `tie_break` 参数影响）**。故探针测出的 35/390 分歧**不含**这条 reloc 机制差异通道。
- **【推测】** 差异量级 <1e-6（单精度噪声），演示批实测 0 分歧、T25 实跑绿，实害极低。但两点需记账：① gap 表"reloc 已一致"不严谨；② 离线探针不能声称 100% 覆盖 sim↔ST 差异面——真正兜底是 **T25 AB 实跑**（本次已复现绿）。
