# 堆垛机驻留点(dwell-point)设计(0708 纳入设计,待实装)

> 触发:对标 deep-research F2(docs/调研档案_0706/对标升级_deep研_0707.md)——dwell-point 选择是
> AS/RS 公认的独立优化维度(Egbelu 1991 / Hwang & Lim 1993 起数十年子文献),我方基线未显式建模。
> 状态:**设计已定、实装待用户拍板下一步后排期**;canonical A10 已登记口径。

## 1. 概念

驻留点 = 堆垛机完成一次作业、下一请求到来前的**空闲停靠位置**。选得好 → 下一次作业期望行程更短。
我方当前隐含假设:每次作业后回 I/O 点 (0,0)(canonical A3「作业起点/终点」+ C1「双程 I/O→仓位→返回」)——
即默认驻留点固定在 (0,0)。这留下一个低成本、高辨识度的优化空间。

## 2. 候选策略(文献经典 4 类)

| 策略 | 规则 | 目标 | 备注 |
|---|---|---|---|
| `io` | 停 I/O 点 (0,0) | 基线 | **我方当前隐含**,作对照锚 |
| `last` | 停最后作业位置(空闲不动) | min 重定位移动 | 单命令场景常用 |
| `center` | 停仓库几何中点 | min 最大行程(minimax) | 无需求信息时的稳健选择 |
| `demand` | 停需求加权期望位置(高频货区重心) | min **期望**行程 | 收益最大,需在库频次信息 |

## 3. 我方落地方案

### 3.1 sim 侧
- 加参数 `dwell_policy ∈ {io, last, center, demand}`;
- cycle time 计算从固定 (0,0) 起算改为**从 dwell_point 起算**,作业毕按策略停靠;
- **收益主场景=出库(取货)**:取货请求 ∝ 频次,驻留点在高频区附近 → 期望取货时间 C3 ↓;
- 对照实验:复用 **30-seed 统计框架**比较 4 策略的 C3(期望取货时间)/C1(双程时间),给收益量化+95%CI;
- 默认 `io` 保回归,升级项开 `demand` 验证。

### 3.2 ST 侧
- PRG_Main 状态机加 **DWELL 态**:作业完成后堆垛机移动到 dwell_point;
- `demand` 策略下 dwell_point = 扫描在库高频货重心(O(n) 轻量,不触看门狗);
- 可视化:堆垛机停驻留点(VisuCraneXY)+ 画面加「驻留点」标记(块3 画面亮点)。

### 3.3 验证
- sim 对照:`demand` vs `io` 的期望取货时间差(量化收益);
- ST:AB 实跑堆垛机空闲移到驻留点 + 可视化正确;
- 一致性:sim↔ST 驻留点位置一致(复用 consistency_probe 思路)。

## 4. 待定项(实装时 / 用户拍板)

1. **默认策略**:`io` 保守 vs `demand` 主推——建议 `demand` 主推 + `io` 对照(呼应我方"主口径+对照"传统);
2. `demand` 期望位置口径:在库高频货重心 vs 历史请求热力重心;
3. **是否进决赛演示**(画面亮点)vs 仅报告分析项;
4. 实时性:`demand` 每次空闲扫在库货算重心,400 库位 O(n) 可行(已初判无压力,实装时实测)。
