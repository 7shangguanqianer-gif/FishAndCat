# 工作站桌面化 · 调研综合与壳定案(0720 deep-research wf_59ff2a32-c4b,105 agents)

## 高置信证据(三票确认,一手源)

1. **file:// CORS**【高,MDN+three.js 论坛】:file:// 下 fetch/GLTF 触发 opaque origin 拦截——本站数据走 `<script>` 注入不受此限(现状 file:// 直开可用),但壳内仍应内置本地 http 服务,消除全部协议差异。
2. **SmartScreen 必弹**【高,微软官方】:未签名 exe 必触发「Windows 保护了你的电脑」,需点「仍要运行」;企业策略可全拦;声誉每个新版本从零。缓解=评委说明文档+zip 分发+右键解锁指引(行业标准做法,无壳能免)。
3. **PyInstaller --onefile 杀软误报是官方认证的重复问题类别**【高,官方 repo 专设标签】+ 运行时解压启动慢 → 若用 PyInstaller,--onedir(绿色目录)更稳。
4. **pywebview+PyInstaller 有未彻底解决的启动崩溃坑**【高,多 issue 复现】:pythonnet/clr_loader「Failed to resolve Python.Runtime.Loader.Initialize」。
5. **WebView2 预装现实**【premise 审计判「存疑」】:Win11 系统自带;**Win10 非普遍预装**(精简/企业镜像可能缺失)——评委机离线+无管理员时,WebView2 缺失=pywebview/Tauri 死局(Evergreen Bootstrapper 要联网;Fixed Version 免安装分发可行但体积 ~200MB≈Electron)。
6. **Electron portable**【高,官方 API+多源印证】:自带 Chromium+Node,50-150MB,零系统 WebView 依赖;loadURL/loadFile 支持多页融合单窗口导航(标准模式)。Tauri 3-15MB 但同吃 WebView2 风险+Rust 新工具链。

## 研究空白(如实):设置持久化目录/窗口记忆/关于窗口惯例未产出确认 claim;Neutralino 维护活跃度未核。Electron 生态的 electron-store/electron-window-state 是众所周知方案(知识判断,非本轮检索结论,PoC 实测为准)。

## 壳定案:**Electron portable**(fable 决策)

评委场景硬约束=免安装+离线+无管理员+双击即用。逐项对比:
- pywebview:被证据 4(冻结崩溃坑)与 5(Win10 WebView2 缺失死局)双雷命中要害——离线场景无法现场补运行时;
- Tauri:同吃证据 5,且 Rust 构建链赛前引入风险;
- **Electron:体积换确定性——自带渲染内核,评委机 100% 能跑**;提交包无体积硬限(用户未勾);SmartScreen 所有方案同样面对,标准缓解即可。
- 后备路径双保险:提交包同时附「浏览器直开 src/00_导览.html」说明(现状 file:// 可用已验证)。

## 落地要点
- electron-builder portable target;--onedir 思想同理(目录形态优先,单 exe portable 亦可测);
- 壳内起本地静态服务(或 loadFile+保持 script 注入数据模式);
- 导航壳:单 BrowserWindow + 导航方案 PoC 定(loadFile 切页 vs 壳页 iframe);
- 设置持久化/窗口记忆:electron-store/electron-window-state,PoC 实测。
