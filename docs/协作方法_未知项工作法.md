# 协作方法:未知项工作法(2026-07-05 定,源自用户提供的 Fable 协作文章)

> 核心思想(转述):给 AI 的提示词是"地图",真实代码库与现实约束是"疆域";
> 二者的差=未知项。工作质量的瓶颈不在 AI 能力,在于**多快把未知项暴露并消掉**。
> 未知项四象限:已知已知(写进提示词的)/ 已知未知(知道自己没想清的)/
> 未知已知(显而易见到不会写下来、但见到就认得的,如视觉口味)/ 未知未知(压根没想到的)。

## 1. 本项目已在用的对应机制(识别,不重造)

| 文章技法 | 我们的现行机制 |
|---|---|
| 假设显式化 | canonical_assumptions.md + 口径矩阵(未知→参数+开关) |
| Interview(问出歧义) | 题面歧义参数化+官方群 B1/B2;拍板清单一次成批问用户 |
| Implementation notes | 实验发现 F 系列(每条含撞墙与偏离)+ 信箱回执格式 |
| 事后盘点未知 | 通俗导览 §7 深度尝试实录(试了什么/怎么撞墙/判词) |
| 偏离协议 | Codex 纪律="遇与预期不符→停下截图汇报"(执行者用保守版) |
| Blindspot pass | 跑算力前双硬门槛(彻查脚本+必须联网查现成方案) |
| Pitch/explainer | 项目通俗导览 + 报告生成器(评委版 explainer) |

## 2. 本次新增的四个机制(0705 定,立即生效)

1. **接管测验**(quiz 用在 AI 交接上):新会话/新账号接手,读完三件套后必须在首条
   回复里自答 5 道核对题(题目在两份接管提示词里,答案全部可从文档找到)。
   答不对=没读懂,用户肉眼可查——防"假接手"。
2. **设计稿先行**(prototype 消"未知已知"):凡是用户会用眼睛验收的产出(可视化画面/
   报告版式/图表风格),先出低成本设计稿(HTML/PNG 示意)给用户 react,批了才施工。
   首个应用:T17 可视化屏幕设计稿 → 用户过目 → 才签发批次 C。
   依据:用户的视觉偏好是典型"未知已知"——说不出来,但看到就知道要不要。
3. **课后小测**(quiz 用在用户身上):每周一课末尾附 5 题小测(课 0 的已附在导览文末),
   全对才算该课通过;错题=下一课重点。这是答辩问答的免疫系统,也是 R2 的验收标准。
4. **周报未知项栏**(blindspot pass 制度化):每周五周报固定栏目"下周未知项清单"——
   每项标注四象限归类+谁来消(我/Codex/用户/官方)。当前存量未知项:
   B1/B2 题面歧义(已知未知,官方消)、L4 监视页读数(已知未知,批次 B 消)、
   用户对可视化画面的口味(未知已知,设计稿消)、提交邮件网关实际行为(已知未知,R3 彩排消)。

## 3. 一条元规则

规格说得太细,AI 该转弯时不敢转;说得太粗,AI 按"行业惯例"猜——两头都输。
解法不是找完美粒度,是**把"遇到未知怎么办"本身写进任务**:哪些事可自行保守处理并记录,
哪些事必须停下来问(本项目:口径/头条数字/用户拍板项必停;实现细节可自行+记录)。

## 4. 查证打法三分类(0705 L7 研究链两轮学费换来的 SOP,用户已采纳)

查证前先判断信息类型,再选打法——**打法错配 = 白烧额度**:

| 信息类型 | 例子 | 正确打法 | 错误打法(实测踩过) |
|---|---|---|---|
| 营销宣称 | "再生回收 60%""节能 70%" | 对抗验证(多票互驳)→ 大概率枪毙,枪毙记录进防伪清单 | 直接引用 |
| 工程参数 | 速度/加速度/装卸时间/MTBF | **定向枪**:厂商规格书、固件手册、招标技术文件(单 agent 窄任务) | 全管线 web 搜索(公开网页没有,付费墙挡住,两轮空手) |
| 理论假设 | 切比雪夫/SC-DC 定义 | 原始论文原文锚(作者-年份-DOI-原句) | 引二手转述 |

全管线 deep-research 只用于"不知道该问什么"的开荒;知道要什么后一律定向枪。
负面出处黑名单(防伪清单)= 与正面引用同等重要的资产,写报告前必过一遍(现存于 L7 设计稿 G8)。

## 5. Agent 任务书标准头(0705 失控树事故后定,用户已采纳)

给任何 agent 的任务书,前四行固定:

```
模型档位:<haiku/sonnet/opus,按"机械活低档、判断中档、综合高档">
禁止使用 Agent 工具启动任何子代理(只许自己用 WebSearch/WebFetch/Read 等干活)
交付格式:纯数据(表格/清单),每条带来源 URL 与原文摘录,不写给人看的散文
来源纪律:官方域名/一手来源优先;找不到就如实说找不到,不许编造不许凑数
```

背景:0705 一次 ABB 搜索任务未写第 2 行,agent 自行展开成 4×5 孙代理树(失控但止损成功);
另,多子代理任务全 Fable 主档曾烧穿会话限额——档位分层是硬纪律,不是优化项。
