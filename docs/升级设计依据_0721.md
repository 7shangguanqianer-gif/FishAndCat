## 1. 算法对比公平性:SPEC 系列运行披露规则【A,跨域类比】

- 一手确证(spec.org 官方规则,多条 3-0/2-1):①同硬件同配置是有效对比前提;②负载注入速率固定不可调;③公布 peak(调优)必须同时公布 base(统一调优);④竞争性对比必须披露比较口径/数据来源/获取日期;⑤被比较结果须符合运行规则且公开可得;⑥jbb2015:两项指标须相邻展示、禁止跨运行类别比较。
- **本域定位:计算系统基准规范→仓储放置算法对比=跨域类比借用,报告措辞「参照 SPEC 系列基准披露惯例」,不称行业标准。**
- 01 页现状与之天然吻合(可写进报告):三 lane 同一到达队列=同数据同负载;SEQ 基线涨跌徽章=baseline delta;30-seed 固定种子=可复现。3 版设计强化:对比区常驻「同源口径声明」;三算法指标相邻展示(jbb2015 惯例)。
- 学术佐证【B】:arXiv 2203.07889 随机占优度量替代均值比较(度量本身 3-0;其配套图形化分解方法 0-3 被杀,**不得引用**)。30-seed 分布带(F5)可引「不只报均值」精神。

## 2. 多准则加权分解:swing-weighting 硬条款 + 呈现层设计自由【A+D】

- 英国政府 Analysis Function MCDA 官方指南【A,3-0】:「Simple weighting is not a valid approach... The approach that should be used is swing weighting」——简单直接赋权忽略准则取值范围,无效。
- **对 01 评分解剖室(F4)的硬约束:S3 SCORE 是固定四权重(score_policy.density_contract「frozen fixed weights」)。呈现时诚实标注「固定权重口径,非 swing-weighting 论证产物」,禁止暗示权重经过 MCDA 官方方法校准。**答辩若被问权重来源,答「竞赛场景固定口径+敏感性分析佐证」(whatif json 正好补此位)。
- 呈现层【D】:瀑布图/堆叠图/雷达图/龙卷风图作为「MCDA 官方推荐可视化」两条主张均被对抗核查推翻(0-3/1-2)——**图式选择=设计自由,报告不称官方惯例**;敏感性分析用龙卷风/whatif 属通行工程实践,措辞「工程惯例」。

## 3. 数字孪生对照:ISO 23247 定义锚 + 呈现层设计自由【A/B+D】

- ISO 23247-1:2021 正式版 PDF【A,2-1】:数字孪生定义核心=物理元素与数字表示之间的**同步**(3.2.3、4.1 两处重复,区别于静态 3D 模型)。
- 同步方法可事件驱动或时间驱动;保真度仅「appropriate level of fidelity」无数值门槛【B,来源含 FDIS 草案镜像,版本措辞留余地】。
- **对 03 对照剧场(C4)的注入:引 ISO 23247「同步」定义框架;分屏/帧带拖动条=标准不规定,设计自由【D】;保真度免责措辞「适当保真度+形态对照非逐帧等价」(诚实口径,呼应仿真≠实拍同 seed 的事实)。**
- 开放缺口:厂商(西门子/达索/ABB)数字孪生展示一手文档本轮未获——如 3 版拍板后需厂商实践佐证再补一轮定向检索,当前按设计自由处理。

## 4. 证据链/审计呈现:ISO/IEC 17025 仅可类比【A(范围界定)+D】

- ISO 官方页【A,3-0】:17025 范围=实验室能力认可,**不覆盖** FAT/SAT 工业验收与 HMI 证据卡/哈希显示。
