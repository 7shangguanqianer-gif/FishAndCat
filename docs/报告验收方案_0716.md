## 验收流程:三遍读,三个视角

### 第一遍 · 评委粗读视角(10 分钟,模拟评委第一印象)
只读:摘要 → 每章首段 → 所有图表标题。合上文档回答三问:
- [ ] 能否用三句话说出这个作品做了什么、好多少、怎么证明的?
- [ ] 头条数字(8.17±0.67s / 61.6% / 75-0)是否在前两页就能看到且带口径标签?
- [ ] 有没有一眼看到"这些数字怎么复核"的入口(查证矩阵/复现命令)?
**任何一问答"否"→ 记入改进项,这是报告周的头等修改。**

### 第二遍 · 评分表对照视角(30 分钟,逐项找证据)
拿官方评分表逐项对照,每项标 ✓(有据)/△(有但弱)/✗(缺):

| 评分项(权重) | 在报告里找什么 | 检验标准 |
|---|---|---|
| 建模准确性(30% 之半) | §1 建模假设章 | 货物画像(重量/频次/体积)、层承重系数 1.0 递减 0.02、预占 sum 口径 133 格、双轴运动学——与题面逐条对应,自设假设与题面给定分开写 |
| 验证结果展示(30% 之半) | §2/§4/§5 | 30-seed 泛化主证+p<0.001 有计算依据、18×8×10 鲁棒性矩阵、98.5%/99.19% 双口径分别标注、负结果(E13 全不可行等)如实呈现 |
| AC500 实现(30%) | §5 PLC 章 | ST 结构化+功能块封装+边界判断(对应题面三条要求)、75/0 证据级别写清"PC 仿真编译+在线回读"、420/420 等价性 |
| 算法创新性(10%) | §3 算法章 | AWRA-LS 增量表述=批次重排+局部搜索(不吹"另一套高级评分"、不称在线学习)——创新点求准不求大 |
| 可视化(20%,决赛) | §6 | 只需检查:与决赛件口径一致、不提前越界宣称 |

### 第三遍 · 诚实性抽查视角(20 分钟,防"数字能复现≠断言有据")
- [ ] 抽 5 个关键数字沿"报告→脚本/CSV"溯源:8.17±0.67(final_test_30seed)、61.6%、98.5%/99.19%(oracle_gap)、75/0(plc_evidence.csv)、p<0.001(lifecycle_stats.py)
- [ ] 口径红线扫一遍:所有 sim 数带标注;9/9 只写"运行成功";F5 写"当前环境负结果";无"完全闭环/400 格孪生/理论上限"字样
- [ ] 加分项对照(西门子杯 7 条):有无"局限性坦诚披露"章?验证三形态占几形态?图表密度够不够?

### 产出与判定
- 每章记录:**过 / 改(具体改什么)/ 疑问(要 Claude 解释)** 三态。
- 全部"过"或"改"项均为报告周可修 → 说「WORD 已验收,PPT 可以重生」;存在原则性"疑问" → 先议再验。
