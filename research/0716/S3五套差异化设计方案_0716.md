# S3 五套差异化设计方案

日期：2026-07-16  
阶段：结构与视觉选型；不是实现稿

## 共同硬约束

- 正常态约 34/66：左算法决策台、右 3D、底部统一遥测；不重复当前动作/货物/步骤。
- 右侧只表现 20×20 逻辑 SIM；54 格 FIO 独立，不出现 AB/FIO 在线暗示。
- 货架/背景白灰建筑化；货物、同侧 T 型并行双链、堆垛机工业细化。
- 货物颜色=重量，边框/动画=生命周期，角标=频次；2D/3D 同义且不只靠颜色。
- `管理货物 0→267` 与 `总占用 133→400` 双计数；400/400 只作静态终点。
- Top3/全矩阵、路径/能耗/吞吐等只有在新 trace 真正导出后才显示。
- 对比模式使用同源输入、event barrier、各自 SIM 时钟；AWRA-LS 单列离线批量参考。

## A · 货物决策链

### 核心隐喻

每件货物形成一条可追溯“决策链”：先看货物画像，再看策略推荐、候选货位、目标位与完整动作证据。它最直接回答“为什么是这个算法、为什么放这里”。

### 信息架构

1. 情景与证据状态；
2. 当前货物画像；
3. 启动时规则推荐与固定策略；
4. 目标位 + Top3 证据；
5. 20×20 小地图；
6. 次级入口：八算法、全矩阵、单件沙盒。

左侧以单列决策卡推进，当前结论始终位于视线中心；Top3 用三行紧凑条形分解，完整矩阵进入二级抽屉。

### 正常态与连续填充

- 每件货物完成时，决策状态从“待决策→已锁位→运输中→已落位”依次推进。
- 0/25/50/75/90/100% 作为书签；第 10 件慢放时展开候选证据，其他货物保持连续快进。
- 左侧只显示当前决策，不堆叠历史卡；历史进入可搜索事件列表。

### 3D 与建模

- 默认三分之四斜上视角，货架占右侧约 70%，双链与载货台占前下部，目标格不被标签遮挡。
- 堆垛机为银灰门架、深灰导轨、浅蓝活动轴、清晰载货台/叉臂；ABB 红只提示当前活动或警报。
- 双链采用灰色滚筒、支腿、挡停、独立入/出库方向；货物从输送线到载货台只有一个可见交接点。
- 轨迹只显示“已走实线 + 下一段虚线”，完成后立即淡出，避免全路径铺满画面。

### 对比态

进入独立 A/B 实验页：每侧上方 3D、下方 2D；中央保留同一货物决策记录和差值条。正常页不同时渲染两套 3D。

### 美术方向

ABB 瑞士式白底、黑灰网格、深蓝决策色、少量红色动作提示；信息密度高但无装饰性英文。

### 优势 / 风险 / 场景

- 优势：最符合“算法为主、3D 为证”；单件解释最强；适合评委追问。
- 风险：267 件可能重复，必须依靠慢放规则和容量书签控制节奏。
- 最适合：算法原理、当前货物、目标位合理性。

## B · 满仓进度仪

### 核心隐喻

页面是一块“容量演进仪表”：主角不是单件货，而是可行空间如何从宽松走向紧张，最终抵达静态满库边界。

### 信息架构

1. 双计数主仪表：管理位 / 总占用；
2. 当前容量阶段与剩余可行位；
3. 当前货物和策略理由；
4. 阶段风险：重货可行层、队列、拒收；
5. 小型 2D 占用演进图；
6. 书签与速度。

左侧上半以容量阶段为骨架，下半才是当前货物和算法，形成“全局→局部”的阅读顺序。

### 正常态与连续填充

- 以 `0/267`、`67/267`、`134/267`、`201/267`、`241/267`、`267/267` 六个稳定 checkpoint 组织叙事。
- 书签间仍逐件消费完整事件；进度环同时显示 133 个 PRE，避免把管理位百分比误说成总仓占用。
- 90% 后自动切入“高密度观察”，放大可行位收缩与真实终止原因。

### 3D 与建模

- 更宽的正立面透视，货架像建筑剖面；高密度后可启用列/层剖切，但不能隐藏库存状态。
- 堆垛机和双链保持精细实体；货架自身弱化为建筑骨架，让货物填充变化成为主视觉。
- 轨迹用短暂“物流脉冲”沿当前路径推进；不保留历史轨迹，历史由容量带表达。

### 对比态

对比的是相同管理位数量时的布局，而不是相同动画秒数。左右 2D/3D 各自停在同一 event index，底部比较剩余可行位、路径、重心和预期取货。

### 美术方向

明亮建筑白灰，容量阶段用冷青→深蓝，当前目标用黄，异常用红；布局更像建筑状态展板而非传统控制台。

### 优势 / 风险 / 场景

- 优势：最强地回答“从 0 到满是否真实连续”“267 与 400 什么关系”。
- 风险：容易退化成填格动画；当前策略理由必须一直可见。
- 最适合：规模、连续性、满库边界、可行域收缩。

## C · 策略裁判台

### 核心隐喻

页面是一座“裁判台”：先说明规则推荐为何选定某策略，再让评委选择两个同信息集算法进行可复现对比。

### 信息架构

1. 情景画像与推荐原因码；
2. “本回放固定策略”而非智能学习；
3. 明显的“查看全部 8 种算法”入口；
4. 当前货物与目标位；
5. 公平性锁：seed、货流、拟真层、预占、hash；
6. 统计证据与代表回放分层。

左侧采用深石墨控制台，白色 3D 场景与其形成强对比；中文结论优先，AUTO/P95/SIM 只作为已解释缩写。

### 正常态与连续填充

- 默认只播放推荐的固定在线策略；情景输入与推荐原因常驻。
- 连续填充保持主流程；每第 10 件展示该策略的实际目标与代价，不显示未导出的候选数值。
- AWRA-LS 入口显式标“离线批量参考”，不进入在线竞赛默认列表。

### 3D 与建模

- 使用固定、对称、可复现的审判镜头；不以夸张运镜制造优劣。
- 堆垛机为中性灰，A/B 差异只编码货物目标和轨迹，不给某算法更醒目的美术待遇。
- 双链、目标位、货物标签在所有算法中完全同构。

### 对比态

整屏变为 A/B 两个 lane：每侧 3D + 2D，中央共享 event barrier，底部只保留真实差值。共享输入 hash 不同、任一算法失效或证据缺失时对比页拒绝启动。

### 美术方向

左侧深石墨、右侧高亮白灰；青/橙作为 A/B 中性色，ABB 红只用于失败/当前动作，避免“红色一方看起来更优”。

### 优势 / 风险 / 场景

- 优势：算法对照、公平性和真实边界最强；“全量 8”入口最清楚。
- 风险：最像 benchmark；必须严格隔离 S1 统计与 S3 回放，也要控制双 3D 性能。
- 最适合：算法创新、对比实验、AUTO 真实含义、限制说明。

## D · 仓位地形图

### 核心隐喻

把 20×20 仓位看作一张“决策地形”：预占、承重、距离、频次和剩余空间共同决定目标格；3D 是地形的实体对应。

### 信息架构

1. 当前货物与硬约束；
2. 20×20 可行空间主图；
3. 胜出位 + Top3；
4. 目标位分项解释；
5. 当前 3D 动作；
6. 全矩阵证据抽屉。

左侧 2D 地图占最大面积；决策卡和指标围绕地图布置。地图点击、Top3 行和 3D 目标格三向联动。

### 正常态与连续填充

- 每次入库前先显示可行/不可行空间，锁位后仅保留胜出位和当前路径。
- 填充过程中强调“可行前沿”如何移动；容量书签按管理位进度，不按总仓百分比。
- 全候选矩阵只在 `candidate_audit_ref` 存在且 hash 正确时开放。

### 3D 与建模

- 略偏正交的 30° 俯视，减小透视畸变，便于 2D 坐标和 3D 格位对应。
- 货架保持白灰建筑体，预占体与柜面齐平；候选只用边框/纹理叠加，不把格子挤出或悬浮。
- 当前轨迹与地图路径使用同一段落编号；3D 只显示当前段，不显示所有候选路径。

### 对比态

两个 2D 地形图居主，两个小 3D 负责动作验证；支持同一货物在 A/B 中目标位不同的 linked highlight。差值先看约束与空间，再看时间/能耗。

### 美术方向

工程蓝图式浅蓝灰网格、白色货架、深蓝/青货物、黄色目标；避免热力图彩虹色和大面积淡绿。

### 优势 / 风险 / 场景

- 优势：解释货位为何合理最直观；2D/3D 同源关系最强。
- 风险：容易喧宾夺主变成矩阵工具；默认必须限制为当前货物和 Top3。
- 最适合：承重、预占、距离、货位选择、候选分解。

## E · 答辩证据剧场

### 核心隐喻

页面按评委问题组织：“为何选此策略？”“为何选此货位？”“过程是否连续？”“证据边界是什么？”；每个问题都有对应的同源 2D/3D 答案。

### 信息架构

1. 三个核心问题与当前答案；
2. 当前情景/货物/目标位；
3. 证据状态：已验证、代表回放、待验证；
4. 3D 作业主舞台；
5. 容量章节与证据快照；
6. 限制说明与独立 54 格证据页链接位。

左侧不做传统参数表，而做问题导航和结论卡；技术细节进入第二层，不在首屏堆满。

### 正常态与连续填充

- 0→267 被编排成“起点、低密度、中密度、高密度、静态满库”五章；章节间连续快进，直接跳转明确写“快照”。
- 当前章节只突出一个问题和一个结论；第 10 件慢放负责展示算法证据。
- 单件沙盒和八算法对照保留，但从问题卡进入，不在首屏占据大块表单。

### 3D 与建模

- 一个经截图验收的固定总览镜头，手动调节后可一键恢复；场景避免自由镜头导致目标位消失。
- 货架立面完整，双链/交接/载货台清楚；用受控聚光和局部描边引导，不用浮动大标签遮挡目标。
- 轨迹只在“过程是否连续”问题下出现；其他章节关闭轨迹，保持画面干净。

### 对比态

根据问题切换证据：策略问题进入 C 式 A/B；空间问题进入 D 式 linked map；连续性问题进入事件带 + 双 3D。它是按问题调度不同证据，而不是一个固定复杂面板。

### 美术方向

展陈式大字号、暖白背景、黑灰结构、ABB 红章节标记；留白有意服务结论，不使用装饰性英文或无意义边框。

### 优势 / 风险 / 场景

- 优势：3–5 分钟答辩最清晰，主动呈现已验证/未验证边界。
- 风险：自由实验效率最低，不应冒充日常工业 HMI；后续教程也容易重复，需要合并设计。
- 最适合：决赛陈述、评委追问、可信度和工程边界。

## 推荐组合（未替用户拍板）

- **主壳推荐 A**：最贴合已锁定的“算法决策为主，三维为证”。
- **吸收 B 的容量书签**：解决 0→267 连续演示与 133/267/400 解释。
- **吸收 D 的 linked 2D/3D**：解决 Top3 和目标位的空间解释。
- **把 C 做成独立对比模式**：不污染正常 34/66，也能完整满足 Q11。
- **E 只做以后可选的答辩模式**：不要与 Q12 的 60 秒教学同时抢首屏。

用户可选择单一方案，也可按“主壳 + 模块”组合；选定前不进入 HTML 实现。
