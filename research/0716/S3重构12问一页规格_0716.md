# S3 重构 12 问一页规格

日期：2026-07-16  
状态：12 问已收敛；只进入设计选型，不授权修改候选或正式 HTML

## 一句话目标

用一个 20×20 逻辑仓储 SIM 页面，让评委先看懂“当前货物为何选该策略与货位”，再由同源 2D/3D 连续作业证明决策；支持 canonical 静态满放演示和公平 A/B 对比，但不把逻辑 SIM、Factory I/O、PLC 或统计证据混成一件事。

## 范围与非范围

**本轮范围**

- 预设情景为默认入口；规则推荐、八算法对照属于情景；单件自定义是独立沙盒。
- 正常态固定约 34/66：左侧紧凑算法决策台，右侧干净 3D，底部一条共享遥测/播放带。
- 货架/背景为白灰建筑模型；货物、同侧并行双链和堆垛机为精细工业抽象；双链与货架正面成 T 型。
- 默认显示胜出位与 Top 3；全候选矩阵主动展开。无 sim 导出证据时禁用，不得前端造数。
- 货物填充色=重量三级，边框/运动=生命周期，角标=频次；2D/3D 同义并有文字/纹理冗余。

**明确非范围**

- 不把 54 格 Factory I/O 嵌入 S3；它由独立“代表性物理 I/O 证据浏览器”承担。
- 不宣称 AB/PLC/Factory I/O 在线、不称 400 格物理孪生、不把 AUTO 写成逐货在线学习。
- 不修改 canonical、报告头条或现有 S1/S3 统计；不混合 S1 多 seed 与 S3 单 seed 数字。
- 60 秒点击式新手教学暂不实施；核心页面截图通过后再正式询问并拍板。

## canonical 满仓合同

- 20×20 共 400 格；官方 `(列+层)%3==0` 预占 133 格，算法可管理 267 格。
- 主页面双计数：`管理货物 0/267→267/267`；`总占用 133/400→400/400`。
- 133 格始终标“题面预占/不可用”，不算算法放入货物；因此不得写“空仓 0/400”。
- `400/400` 是静态容量边界。现有混合作业先入后出，满库后无法继续接收入库；如需满库循环，必须另建先出后入或缓冲区模型。
- 新增独立 `fill-only` trace，不裁剪现有 120 件初始库存 + 100 paired IN/OUT 回放。
- 主线目标 `FILL_CANONICAL_SAFE` 需冻结 profile、seed、货序、权重和归一化，正式证明 267/267、0 fail、0 viol 后方可进页面；267 只是槽位上限，不是任意货流/策略已验证能力。
- 可选 `FILL_NO_PRESET_400` 只作 `sim-only / 非题面口径 / 容量反事实`，与 canonical、PLC、FIO、头条 KPI 完全分离。

## 连续播放合同

- 每件货完整经历入口→决策→输送→交接→双轴→叉臂→落位→空载回位；只在 `RACK_STORED` 后库存 +1。
- SIM 时间不改；演示倍率只做单调映射。长排队可强压缩，设备运动按真实距离/轴速保留相对时长和最低可视时长。
- 第 1 件、每第 10 件、25/50/75/90/100% 边界、关键决策、拒收/终止前自动慢放。
- 快捷点为 `0/25/50/75/90/100%` 管理位进度。连续快进仍消费全部事件；直接定位必须标“快照跳转”，只落到货物已落位且设备回位的稳定 checkpoint。

## 公平对比合同

- 主模式：同一情景、不同在线算法；次模式：同一算法、不同情景。
- 同算法对比必须共享货物 ID/属性/顺序、seed、到达、噪声、故障流、拟真层、预占规则、权重和运动参数；共享输入 hash 不一致即 fail-closed。
- 默认按 `event_id / 第 n 件` 设置叙事屏障；A/B 各保留自己的 SIM 时钟，先完成者只进入展示等待，不篡改 KPI。
- 每个 lane 同时拥有自己的 2D 图和 3D 场景，共用事件轴和差值条；一侧失败后停在真实终态，禁止自动换算法、调序或换货。
- AWRA-LS 是离线批量 Rank→Assign→局部搜索，只能作为“离线批量上界/参考”，不与在线策略宣传实时公平胜负。

## 指标合同

1. 结果/安全：成功落位、失败、拒收、违规、终止原因、管理位/总占用。
2. 时间/距离：累计双程时间 `s`、双程路径 `m`、makespan `s`、响应 P50/P95 `s`。
3. 布局：期望取货 `s`、高频前 20% 均时 `s`、重货前 20% 均层、重心 `m`、利用率。
4. 资源：垂直势能代理 `kg·m`、设备忙时/利用率、队列峰值、故障/停机；`kg·m` 不是 kWh。
5. 差值：绝对差和百分比差；失败域、分母 0、样本不齐时 `N/A`。

当前 S3 未导出的路径、`kg·m`、吞吐不得先画。吞吐只按真实成功完成操作数/makespan 派生；有 fail/viol 或计数口径不齐时显示 `N/A`。禁用“完美摆放比例”；使用入库成功率、零违规、受限 VBS attainment 或同实例真实 LAP gap。

## 数据与实现顺序

1. 修正容量 oracle 守门：LAP 在 `n>m` 时必须拒绝，并校验返回分配数等于货物数。
2. 新建版本化 `s3-fill-comparison-v1`：`shared_input`、独立 lanes、events、stable checkpoints、source/config/result hashes。
3. 预注册并验证 fill profile 与主对比算法；SCORE/AUTO 未通过 267 件在线 fail-closed 门前不得作为满放主演示。
4. 扩展候选审计分块：合法性、拒绝原因、原始值、归一化、权重贡献、排名、平局键、hash；UI 默认只取胜出位 + Top 3。
5. 用单一 reducer 驱动 2D、3D、计数和标签；同一 gid 每帧唯一，owner 切换世界坐标连续。
6. 用户选定五方案之一或混合模块后，才允许改候选页；正式版仍需单独授权。

## 放行标准

- 事实：133 PRE 固定；主线 267 个连续 `RACK_STORED`；最终 267/267 与 400/400；无隐藏 fallback。
- 同源：2D、3D、计数、Top3、指标均来自同 event/source hash；无第二套前端业务状态。
- 连续：每个 phase 边界与中点无瞬移、重复货、穿地或 owner 双持；快跳 reducer 等于从 0 重放。
- 公平：两 lane 共享输入 hash 相同；信息集不同的算法明确分层；失败后不继续宣称可比。
- 口径：常驻 `SIM` 与证据层标签；不加载 FIO/AB 状态；不显示未导出数字。
- 体验：5 秒内看懂当前情景、推荐策略、货物、目标位和下一动作；中文主标签，无重复任务卡；默认镜头同时看见双链、交接、载货台和目标格。
- QA：既有 45 条回放和旧契约继续通过；新 trace 有独立 validator；多视口截图由人工逐图复核，自动绿不能替代画面验收。
