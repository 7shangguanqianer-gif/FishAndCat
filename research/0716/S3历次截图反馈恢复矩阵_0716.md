## 2. 下一版截图门（任何一项失败都不得给用户选）

1. **重复门**：顶部/底部同一作业事实不可重复；DOM 与文案双重扫描。
2. **同源门**：bookmark、2D map、3D inventory、target、cargo owner 的 event id 必须一致。
3. **零起点门**：0/267 时 3D 无管理货物、2D managed=0；133 预占仅表现不可管理体积。
4. **满终点门**：267/267 时 3D managed=267、2D managed=267、total unavailable=400，违规=0。
5. **拓扑门**：同侧并行双链、T 型垂直、单巷道单排；顶视和默认视角均通过。
6. **可读门**：默认镜头看全，标签无遮挡，三色货物/轨迹/机械结构可区分。
7. **真实性门**：七步时长、路径与所有权来自 trace/runtime，不用 CSS 假进度。
8. **保真门**：未触及区域与成熟母版结构一致；正式页不被候选覆盖。

## 4. 独立反例审计新增项（0716）

只读审计没有把“页面能跑”当成通过，新增以下阻断项：

1. **旧 QA 真相源错误**：现有截图脚本仍把“同轴、两侧分段”作为 PASS 条件；在 R14/R15 改成同侧并行 T 形双链前，必须先改断言，不能让测试替错误设计背书。
2. **fill loader 缺失**：当前成熟母版仍固定运行 120 件 mixed trace。书签不得再次先于 loader 出现；必须由 active release `08a7b166031f0856fc77` 的同一 reducer 同时驱动 2D、3D、目标、随载货和容量。
3. **路径可见性不足**：未来段 opacity `.09`，入库线与桅杆蓝过近；路径集合虽完整，但当前截图仍可能视觉消失。下一候选须分离机械结构、入库、空载、出库四种角色色并做像素对比检查。
4. **时长口径仍不够严格**：现有 mixed 动画含固定展示时长与上下限，不是全段严格按物理时间同比。fill 候选须公开“SIM 时长 / 演示时长”，只允许单调压缩，不能反转各阶段长短关系。
5. **浮签语义漂移**：旧 mixed 模式会把黄色浮签切为“出库源位”；fill 候选只允许当前入库目标，故障/复位说明不得漂浮遮挡主场景。
6. **规则重复实现**：预占规则目前在 2D 与 3D 各写一次 `(col+tier)%3==0`。候选 loader 要输出一个容量掩码，2D/3D 共读，避免后续一边改 sum、一边仍画旧规则。
7. **当前货物候选证据缺口**：SCORE 只显示权重，尚未展示当前事件的候选位 Top-N 分项/总分；在接入 audit 前不得伪造排名。
