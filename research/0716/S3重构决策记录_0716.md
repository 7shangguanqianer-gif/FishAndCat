## DR-S3-10 · canonical 满仓定义与 267/400 关系

- 用户选择：`Q9-A 为主 + D 快速跳段`，并追问“为什么只能到 267，400 格能否填满并测试”。
- 裁决：**官方主线不是只有 267 格，而是 133 个题面预占格 + 267 个算法可管理格 = 总占用 400/400。**
- 回源：20×20 共 400 格；官方 `sum` 规则 `(col+tier)%3==0` 预占 133 格，因此算法可用位是 `400-133=267`。[warehouse_sim.py:130](F:/abb_wh_work/sim/warehouse_sim.py:130) [B1B2官方答复_影响分析.md:8](F:/abb_wh_work/docs/B1B2官方答复_影响分析.md:8) [03_Functions.st:29](F:/abb_wh_work/plc/03_Functions.st:29)
- 页面必须同时显示两组数，不能只写“0→满”：
  - `管理货物：0 / 267 → 267 / 267`；
  - `总占用：133 / 400 → 400 / 400`，其中 133 始终以“题面预占/不可用”纹理呈现，不伪装成算法放入的货物。
- 术语：主线称“管理货物从 0 件连续填满全部可用位”，不得称“空仓 0/400”，因为起点已有 133 个预占状态。
- `400/400` 只表示静态容量边界。当前混合作业是先入库再出库，满库时没有空位承接下一件；因此不得写成“满库仍可连续一进一出”。要在满库继续作业，必须另行设计“先出后入”或实体缓冲区并单独验证。
- 当前 S3 仍是 120 件初始库存、100 个 paired IN/OUT 周期的单 seed 代表回放，初末均不是满库；它不能作为本条验收证据。
- 终点只有在 267 件全部成功落位时才能写“全部可用位已填满”；若提前无可行位、输入耗尽或约束拒收，必须显示真实终止原因。

## DR-S3-13 · 两类公平对比与同步规则

- 用户选择：`Q11` 要求先选择对比及目标，再让 2D/3D 货物图与消耗数据平行对比。
- 主对比：**同一情景、不同算法**。锁定同一 `scenario_id`、货物 ID/属性/顺序、seed、到达流、噪声/故障流、拟真层、预占规则、权重与运动参数，唯一变量是算法；AUTO 只能描述为“启动时一次规则推荐”，不能写成逐货在线学习。
- 次对比：**同一算法、不同情景**。使用同一 seed 体系和参数版本，但明确两侧货流可以不同；它回答鲁棒性/敏感性，不得解释为算法间因果优劣。
- 回放默认按 `event_id / 第 n 件货物` 设置同步屏障：A、B 两侧完成第 n 件后才共同进入 n+1；两侧保留各自 SIM 时钟，因此慢的一侧不会被伪装成同速。
