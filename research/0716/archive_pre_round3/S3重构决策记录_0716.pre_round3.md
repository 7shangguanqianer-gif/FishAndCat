## DR-S3-04 · 20×20 与 54 格的展示关系

- 用户选择：`Q4-B`
- 决策：**S3 主页面只保留 20×20 逻辑 SIM，不出现 Factory I/O 54 格视觉。**
- 54 格另建独立 HTML，由 Claude Code 执行候选页任务。
- 54 格页面的意义：
  1. 证明代表性物理设备、传感器、执行器和 I/O 时序，而不是补足 400 格规模；
  2. 说明逻辑货位命令如何落到 Entry、Load、叉臂/Lift、Rack、Unload、Exit；
  3. 展示真实本地截图、点表和证据，而不是再造一个伪 3D；
  4. 常驻呈现未验证边界，避免被理解为 20×20 几何孪生或完全闭环。
- 推荐页面定位：**“54 格代表性物理 I/O 证据浏览器”**，不叫数字孪生，不承载 20×20 算法 KPI。

## DR-S3-06 · 混合建模语言

- 用户选择：`Q6 主推 C；货物、入库/出库链、运货机采用 A`
- 决策：
  - 20×20 储货柜/空间背景采用建筑模型风：白灰、低饱和、干净平面关系；
  - 货物、双输送链、堆垛运货机采用精细工业抽象：结构、轴向、叉臂、滚筒和交接关系清晰；

## DR-S3-07 · T 型同侧并行双链

- 用户选择：`Q7 同侧并行双链；推荐 Factory I/O 形式；与储货柜成 T 型垂直`
- 决策：
  - 顶视图中，双输送链与储货柜正面垂直，构成清晰 T 型；
  - 入库链和出库链同侧、并行、同一水平基准，各自有独立滚筒与停止/交接位；
  - 两链不得交叉、共带或在同一空间生成货物；
  - 堆垛机载货台与两条链之间设置单一、可见、不会跳变的交接区；
  - 造型可借鉴 Factory I/O 的灰色滚筒、支腿、挡停与安全间距，但页面仍只标逻辑 SIM。
- 验收视角：默认镜头必须同时看到两条链、交接区、载货台和目标货位；不能再靠轨迹线才能区分输送线。
