## 2. 新建可视化对象

1. 设备树里右键 **Application → 添加对象(Add object)→ 可视化(Visualization…)**,
   名字填 `VisuMain` → 添加(Add)。
   - AB 会自动同时创建 **Visualization Manager(可视化管理器)** 和 VISU_TASK 任务,不用管它们。
2. 双击 VisuMain 打开画面编辑器:左边是画布,右边是 **属性(Properties)** 面板 +
   **工具箱(ToolBox)**(常用分类:基本(Basic)/ 常用控件(Common Controls))。
   - 找不到属性面板:菜单 视图(View)→ 属性(Properties)。
3. 可选:属性里把画面大小设为 1280×720(录屏 16:9 友好)。

## 3. 仓位网格(核心步骤,全卡最重要的 10 分钟)

**3.1 画模板格子**:工具箱 基本(Basic)→ 矩形(Rectangle),在画布随便画一个,然后在属性面板精确设置:

| 属性(英文界面名) | 值 |
|---|---|
| 位置 Position → X / Y | 40 / 40 |
| 位置 Position → Width / Height | 28 / 20 |
| 颜色 Colors → 线条 Line/Frame color | 白色或浅灰(格子间隙感) |

**3.2 绑颜色变量(手工输入占位符)**:属性面板 →
**颜色变量(Color variables)→ 正常状态(Normal state)→ 填充色(Fill color)**,填:

```
GVL_Visu.VisuSlotColor[$FIRSTDIM$, $SECONDDIM$]
```

`$FIRSTDIM$`/`$SECONDDIM$` 是 CODESYS 官方的数组占位符,下一步的倍增命令会把它们
自动替换成 0..19 的下标——这就是不用手工绑 400 次的关键。

**3.3 倍增成 20×20**:右键选中这个矩形 → **倍增可视化元件(Multiply visu element…)**
(若右键菜单没有,看顶部菜单 可视化(Visualization)里找同名命令)。对话框里:

- 第一维(FIRSTDIM,横向=列):元件数量 **20**,起始索引 0,元件间距/偏移 = **28**(等于格宽,无缝);
- 第二维(SECONDDIM,纵向=显示行):数量 **20**,起始索引 0,偏移 = **20**(等于格高);
- 对话框有预览,确认是 20×20 紧贴网格再点 OK。字段名与上述对不上时:**截图停下汇报,别猜**。

> 为什么第二维叫"显示行"不叫"层":画面第 0 行在最上面,而仓库第 0 层在最下面。
> **上下翻转已在 PLC 侧做掉**(FB_VisuRefresh:显示行=19-层),画面直接绑就是对的。

**3.4 行列标注(静态文本,量少手工)**:工具箱 基本 → 文本域(Text field),
去掉边框(Colors → 无填充/无线条),Texts → Text 填数字。
- 网格下方一行:列号 `0`(x≈48)、`5`(x≈188)、`10`(x≈328)、`15`(x≈468)、`19`(x≈580),y=445;
- 网格左侧一列:层号 `19`(y≈42)、`15`(y≈122)、`10`(y≈222)、`5`(y≈322)、`0`(y≈422),x=12;
- 网格下方 y≈460 加一行小字:`列0~19 →   层0在底层(承重最大),I/O口在左下角`。

**3.5 图例(6 个静态色块+文字,放网格下方 y≈480)**:每个=一个小矩形(16×12,**静态填充色**,
不绑变量)+右侧文本。颜色值(与 FC_StateToColor 完全一致):

| 含义 | 静态填充色(Colors→Fill color 里输) |
|---|---|
| 空闲 | 16#E8E8E8 |
| 预占用 | 16#707070 |
| 已占用 | 16#43A047 |
| 推荐 | 16#FFC107 |
| 路径 | 16#42A5F5 |
| 报警 | 16#E53935 |

## 4. 堆垛机图元(会动的橙色圆)

1. 工具箱 基本(Basic)→ 椭圆(Ellipse),画一个;属性:
   - **Position → X=0, Y=0**(必须放在画布左上角原点!)、Width=28, Height=20;
   - Colors → Fill color = 16#FF6D00(橙),Line = 深橙或黑。
2. 属性 → **绝对移动(Absolute movement)→ 移动(Movement)→ X**:填 `GVL_Visu.VisuCraneXpx`;
   **Y**:填 `GVL_Visu.VisuCraneYpx`。
   - 原理:元件从设计位置 (0,0) 被变量平移 (Xpx,Ypx),PLC 输出的就是绝对像素坐标,
     所以设计位置必须是 (0,0),否则会整体偏移。
3. 在线后它应该停在网格**左下角**(I/O 口 = 列0层0)。

## 5. 右侧面板(x≥620 区域)

**5.1 状态行(顶部)**
- 文本域:Position(620,40,560×30);Texts → Text 填 `状态: %s`;
  **文本变量(Text variables)→ 文本变量(Text variable)** 填 `GVL_Visu.VisuStatusText`。
- 报警灯:工具箱 常用控件(Common Controls)→ **灯(Lamp)**,放 (1200,38) 32×32,
  变量(Variable)= `GVL_Visu.VisuAlarm`(TRUE=亮;样式选红色灯)。

**5.2 货物输入区(5 个输入框,y从90起,每行高32)**
每行 = 静态标签(文本域,如 `编号`)+ 输入框。输入框做法(以"重量"为例):
- 文本域,Texts → Text 填 `%.1f`(编号用 `%d`,名称用 `%s`);
- 文本变量 → Text variable = `GVL_Visu.InGoodWeight`;
- **输入配置(Input configuration)→ OnMouseDown/单击 → 写入变量(Write a Variable)**,
  变量同上(有的版本叫"Input configuration → Write variable",勾上即可)——
  在线时**单击输入框会弹出小键盘/输入行**,这就是决赛要求的"交互界面输入货物属性"。
- 五个框依次绑:`InGoodId`(%d)/`InGoodName`(%s)/`InGoodWeight`(%.1f)/`InGoodFreq`(%.1f)/`InGoodVolume`(%.2f)。

**5.3 命令按钮(工具箱 常用控件 → 按钮 Button)**
按钮统一做法:Texts → Text 填中文标签;**输入配置(Input configuration)→
按下变量(Tap variable)** 绑对应命令(按下=TRUE 松开=FALSE,程序里做了沿检测,放心连点):

| 按钮文字 | Tap variable | 位置建议 |
|---|---|---|
| 添加货物 | GVL_Visu.CmdAddGood | (620,270) 100×34 |
| 载入演示批 | GVL_Visu.CmdLoadDemo | (730,270) 100×34 |
| 复位 | GVL_Visu.CmdReset | (840,270) 80×34 |
| 运行分配 | GVL_Visu.CmdRunAssign | (620,380) 100×34 |
| 动画回放 | GVL_Visu.CmdAnimate | (730,380) 100×34 |

**5.4 策略选择(4 个按钮+1 个显示框,y≈320)**
- 4 个按钮:Texts 分别填 `顺序`/`就近`/`评分`/`AWRA-LS`;每个用
  **输入配置 → OnMouseClick → 执行 ST 代码(Execute ST-Code)**,代码分别填:
  `GVL_Visu.SelStrategy := 0;` / `:= 1;` / `:= 2;` / `:= 3;`
- 旁边放一个文本域显示当前值:Text `当前策略: %d`,Text variable = `GVL_Visu.SelStrategy`
  (0=顺序 1=就近 2=评分 3=AWRA-LS,可再加一行静态小字注明)。

**5.5 路径信息(y≈430,三个文本域)**
- `目标: (%d` + 第二个 `, %d)` 分别绑 `VisuTargetX`/`VisuTargetY`(或合成两个框省事);
- `路径长度: %.1f m` ← `GVL_Visu.VisuPathLen`;
- `行程用时: %.2f s` ← `GVL_Visu.VisuExecTime`。

**5.6 统计面板(y≈500 起,每行28,六行)**

| Text | Text variable |
|---|---|
| 入库总时间: %.1f s | GVL_Visu.VisuTotalTime |
| 平均单件: %.2f s | GVL_Visu.VisuAvgTime |
| 空间利用率: %.3f | GVL_Visu.VisuSpaceUtilization |
| 路径总长(损耗): %.0f m | GVL_Visu.VisuWearIndex |
| 约束违规: %d(必须=0) | GVL_Visu.VisuViolationCount |
| 状态码对照(静态小字) | 无变量,内容见下表 |

**5.7 权重调节区(R6 加装,决赛演示杀手锏;y≈660 一行四框)**
做法与 5.2 输入框完全相同(文本域+Text variable+Input configuration→Write variable),四个框:

| 标签 | Text | Text variable(在 GVL_Param,不是 GVL_Visu) |
|---|---|---|
| α效率 | %.2f | GVL_Param.rAlpha |
| β稳定 | %.2f | GVL_Param.rBeta |
| γ能耗 | %.2f | GVL_Param.rGamma |
| δ预留 | %.2f | GVL_Param.rDelta |

旁边放静态小字:`默认 1.0 / 0.6 / 0.4 / 0.15,演示后请改回`。
原理:评分权重本来就是 HMI 可调的全局变量,在线改了**下一次推荐/分配立即生效**——
这就是演示第 7 步"评委现场改权重"的底座。

**状态码中文对照(静态文本,一块小字区;变量字符串必须保持 ASCII 是编码红线,画面负责翻译)**:

```
READY_PRECOUNT=n 就绪(预占n格)   DEMO_LOADED_N=n 演示批已载入
SUGGEST_X=a_Y=b 推荐仓位(a,b)    ASSIGN_RUN_PLACED=n 分配中
IMPROVE_R=r_SWAPS=s 局部搜索中    DONE_OK 全部入库成功
DONE_FAILED_N=n n件无可行位       ALARM_NO_FEASIBLE_SLOT 无可行仓位!
ERR_NEG_INPUT 属性不能为负        ERR_GOODS_FULL_400 货物表已满
ERR_NO_GOODS 请先添加/载入货物    RESET_INIT 已复位
```

## 6. 六步演示脚本(录屏照这个走,含讲解词)

登录运行(Simulation → Login → Start),画面自动初始化。

| 步 | 操作(按钮级) | 画面预期 | 讲解词(一句) |
|---|---|---|---|
| 1 初始状态 | 什么都不按 | 400格网格,49个深灰预占格,状态 READY_PRECOUNT=49,橙圆在左下角 | "20列20层400仓位,题面预占用已标出,I/O口在左下角" |
| 2 轻高频货 | 输入 编号901/名称A/重量5/频次95/体积0.3 → 添加货物 | **低层近I/O口**出现黄格+蓝色L形路径,状态 SUGGEST_X=.._Y=.. → 按 动画回放,橙圆沿蓝路径走过去 | "高频轻货,算法自动推荐离入口最近的低成本位" |
| 3 重货 | 输入 编号902/重量95/频次10/体积0.5 → 添加货物 | 黄格出现在**底层**(层0附近) | "95kg重货只能进低层——承重逐层递减2%的约束在起作用" |
| 4 批量分配 | 复位 → 载入演示批 → 策略选 评分 → 运行分配 | 状态飞过 ASSIGN_RUN_PLACED=n → DONE_OK,20个绿格铺开;**记下"入库总时间"** | "20件演示货在线评分逐件入库,统计面板给出总时间/利用率/违规=0" |
| 5 算法对比 | 复位 → 载入演示批 → 策略选 AWRA-LS → 运行分配 | DONE_OK;**入库总时间比第4步小**,违规仍=0 | "批量模式先全局排序再分配加局部搜索,总时间进一步下降——这就是双层架构的价值" |
| 6 边界报警 | 输入 编号903/重量200/频次50/体积0.5 → 添加货物 | 状态 ALARM_NO_FEASIBLE_SLOT,**红色报警灯亮** | "200kg超过所有仓位限重,系统拒绝并报警——边界体系有23个自测用例兜底" |

第 7 步做完**务必把 δ 改回 0.15**(旁边小字有默认值);此演示不动任何头条数字口径(在线改的是 PLC 运行值,断电即恢复源代码默认)。

加分收尾:再按一次 复位,画面回到第1步状态——"整个演示可无限重放,评委可亲手操作"。

## 9. 常见坑

| 症状 | 原因/解法 |
|---|---|
| 右键没有"Multiply visu element" | 先确认选中的是矩形本体(不是画布);再到顶部 可视化 菜单找;都没有→截图停下汇报(别手工复制400个,等裁决) |
| 格子全是一个颜色/不变色 | 颜色变量必须填在 **Color variables→Normal state→Fill color**(不是静态 Colors);再登录在线看 `GVL_Visu.VisuSlotColor[0,19]` 是否≠0(=0 说明 FB_VisuRefresh 没被调用,检查第1步 PRG_Main 是否贴了最新版) |
| 输入框点了没反应 | 必须 Login+Start 在线状态;检查 Input configuration→Write variable 是否配了 |
| 橙圆位置不对/飘出网格 | 椭圆设计位置必须是 (0,0)(见 §4 原理);检查绑的是 VisuCraneXpx/Ypx 不是 VisuCurrentX/Y |
| 动画按了不动 | 正常:必须先"添加货物"得到一条路径(VisuPathCount>0)才有可放的动画 |
| 部分中文标签显示异常 | 静态文本支持中文;若个别字体发虚,在元件 Texts→Font 换 微软雅黑 |
| 复位后堆垛机不在左下角 | 用的是旧版 PRG_Main——重贴 plc\05(补丁里复位会归位) |
